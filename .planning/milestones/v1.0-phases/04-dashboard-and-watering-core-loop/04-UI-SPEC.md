## Spacing Scale

Standard 8-point scale. All values are multiples of 4. Source: Tailwind v4 default spacing, applied via utility classes in existing components.

| Token | Value | Usage in Phase 4 |
|-------|-------|-----------------|
| xs | 4px | Icon gaps (`gap-1`) |
| sm | 8px | Compact inline spacing, snooze pill gaps (`gap-2`) |
| md | 16px | Card internal padding (`p-4`), card grid gap (`gap-4`) |
| lg | 24px | Section spacing (`space-y-6` on page, `my-8` separator) |
| xl | 32px | Dashboard section gap (`space-y-8`) |
| 2xl | 48px | — (not used in this phase) |
| 3xl | 64px | — (not used in this phase) |

Exceptions:
- Touch targets: 44px minimum height enforced on `WaterButton` (11×11 = 44px), snooze pills (`min-h-[44px]`), and watering history entries (`min-h-[44px]`). This is a WCAG SC 2.5.5 requirement carried from Phase 7.
- `WaterButton` is 44×44px (h-11 w-11) to meet touch target minimum.
- Leaf icon container: 40×40px (h-10 w-10) — intentional sub-scale for visual balance.
- **6px badge icon gap (`gap-1.5`):** Urgency badges use `gap-1.5` (6px) between the 12px status icon and its label. This is a deliberate optical exception — 4px (`gap-1`) is too tight for icon+label legibility at 12px icon size; 8px (`gap-2`) creates excess separation. The 6px value is verified in existing code (`dashboard-plant-card.tsx` badge pattern). Do not normalize to `gap-1` or `gap-2`.
- **6px snooze pill vertical padding (`py-1.5`):** `InlineSnoozePills` uses `py-1.5` (6px) to achieve the necessary vertical hit area within the `min-h-[44px]` constraint while keeping the pill visually compact. 4px (`py-1`) undershoots the touch target; 8px (`py-2`) makes pills appear oversized relative to the badge row. The 6px value is verified in existing code (`dashboard-plant-card.tsx` snooze pill pattern). Do not normalize to `py-1` or `py-2`.

## Component Patterns

### Dashboard Card (`DashboardPlantCard`)

Layout: horizontal flex row with 4 children:
1. Leaf icon avatar — 40×40px rounded-md, `bg-accent/10`, `text-accent`
2. Plant info block — flex-1, text-center: nickname (16px semibold), species (14px muted), room (12px muted)
3. Urgency badge — shrink-0, varies by urgency (see Badge Variants below)
4. `WaterButton` — 44×44px ghost icon button, rounded-full, `bg-accent/10`

Optional 5th row (snooze pills): rendered below card body for overdue/due-today/upcoming-due-today cards only.

Card hover: `hover:shadow-sm hover:border-accent/40 transition-shadow`

Card fade-out animation on water: `motion-safe:transition-all motion-safe:duration-300 opacity-0 scale-95`. The `motion-safe:` prefix is mandatory — do not remove.

### Badge Variants

| Urgency | Variant | Classes |
|---------|---------|---------|
| Overdue | custom | `bg-destructive/10 text-destructive border-destructive/20` + `AlertTriangle` icon |
| Due today | custom | `bg-accent/15 text-accent border-accent/20` + `Droplets` icon |
| Upcoming | outline | `variant="outline"` + `Clock` icon + "In {N}d" label |
| Recently Watered | secondary | `variant="secondary"` + `CheckCircle2` icon + relative time |

Badge icon+label spacing uses `gap-1.5 items-center`. The 6px gap is a documented exception to the 8-point grid — see Spacing Scale Exceptions above.

### Snooze Pills (`InlineSnoozePills`)

Three pills: "1d", "2d", "1w". Each: `rounded-md px-3 py-1.5 text-xs font-medium text-muted-foreground bg-muted/60 hover:bg-muted hover:text-foreground transition-colors min-h-[44px]`.

The `py-1.5` (6px) vertical padding is a documented exception to the 8-point grid — see Spacing Scale Exceptions above.

### Section Headers (`DashboardSection`)

`h2 text-base font-semibold` with muted count in parentheses: `text-muted-foreground font-normal`. Separator above each subsequent section (`my-8`).

Three section titles (exact strings):
- "Needs water" — overdue + due today + upcoming with daysUntil === 0
- "Upcoming" — upcoming with daysUntil > 0
- "Recently Watered" — recently watered group

### Responsive Grid

`grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`

Breakpoints: 1 column (mobile, <640px), 2 columns (sm, ≥640px), 3 columns (lg, ≥1024px).

### Loading Skeleton (`DashboardSkeleton`)

Two skeleton sections, each with 3 skeleton cards. Each card: `flex items-center gap-4 p-4`. Skeleton dimensions: section label `h-5 w-24`, avatar `h-10 w-10 rounded-md`, title `h-4 w-32`, subtitle `h-3 w-20`, badge `h-5 w-16`.

### "All Caught Up" Banner

`flex items-center gap-2 rounded-lg bg-accent/10 px-4 py-2`. Contains `CheckCircle2` (h-5 w-5, text-accent) and `text-sm font-medium` message. No animation — static display.

### Log Watering Dialog (`LogWateringDialog`)

Dual-mode (log / edit). Uses `ResponsiveDialog` — Dialog on desktop, Drawer on mobile. Max width `sm:max-w-[28rem]`. Fields: Date (Calendar popover, defaults to today, future dates disabled) and Note (Input, optional, 280 char max). Primary action button: `bg-accent text-accent-foreground hover:bg-accent/90`.

### Watering History Entry (`WateringHistoryEntry`)

Row: `flex items-start justify-between py-2 min-h-[44px]`. Left: date (`text-sm font-medium`) + middle dot + relative time (`text-muted-foreground`), optional note below (`text-sm text-muted-foreground truncate`). Right: kebab `MoreVertical` icon button (h-8 w-8 ghost).

> **Deviation from D-12 — Delete confirmation dialog:**
> CONTEXT.md decision D-12 states "Delete is immediate with no confirmation dialog." The Phase 7 implementation added an `alert-dialog` confirmation flow ("Delete watering log?" / "Keep log" / "Delete log") for `WateringHistoryEntry`. This is a deliberate deviation from D-12: a watering log deletion recalculates `nextWateringAt` for the plant, making it a consequential and difficult-to-reverse action. The Phase 7 implementer chose to protect against accidental deletion with a confirmation step. The existing code and copy are verified and should be preserved. D-12's "immediate delete" intent applies to the UX framing (no separate confirmation *page*), not to the inline `alert-dialog` safety gate.

## Interaction Contract

### Watering Action (UIAX-05 — Optimistic UI)

1. User taps `WaterButton`.
2. Button immediately enters disabled+loading state (`Loader2 animate-spin`).
3. Plant card begins fade-out (`opacity-0 scale-95` with `motion-safe:duration-300`).
4. `useOptimistic` updates `DashboardClient` state — plant moves to top of "Recently Watered" section client-side.
5. `logWatering` Server Action executes in background.
6. On success: success toast appears; `revalidatePath` refreshes server data; optimistic state confirmed.
7. On failure: error toast with "Try again" intent; optimistic state rolls back automatically.
8. On duplicate: duplicate toast; card re-appears in original section.

### Water Button States

| State | Visual |
|-------|--------|
| Idle | `Droplet` icon, `bg-accent/10` |
| Loading | `Loader2 animate-spin`, button disabled |
| Disabled (demo) | button disabled, snooze shows demo error toast |

### Snooze Pills

Appear on cards in "Needs water" section (overdue, due today, upcoming-due-today). Three pills: 1d, 2d, 1w. Click triggers `snoozeReminder` action and shows success/error toast. Pills are disabled during pending state.

### Sort Order Within Sections

Per CONTEXT.md `Claude's Discretion`:
- Needs water: overdue (most days late first) → due today (alphabetical) → upcoming due today (alphabetical)
- Upcoming: soonest first (ascending `daysUntil`)
- Recently Watered: most recently watered first (descending `wateredAt`)

### Timezone Handling

`TimezoneSync` client component runs `useEffect` once on mount, writes `user_tz` cookie. Server reads cookie in `DashboardContent` to compute `todayStart`/`todayEnd` in UTC. On first visit (no cookie): server defaults to UTC; `TimezoneWarning` banner appears. No manual timezone selection. No further action from executor required.

| State | Trigger | Visual |

| Duplicate blocked | Server returns DUPLICATE sentinel | Informational toast (not error variant) |

## Accessibility Constraints

These are carried over from Phase 7 (UIAX-01 through UIAX-04, all Complete) and must be preserved:

- All icon-only buttons have `aria-label` (e.g. `aria-label="Water {plantNickname}"`)
- All status icons use `aria-hidden="true"` — status is also conveyed by badge text
- Dashboard `h1` is `tabIndex={-1}` for focus-after-navigation
- Section headers use `h2` semantic element
- `motion-safe:` prefix on all CSS transitions — removes animation for `prefers-reduced-motion`
- Minimum 44px touch target on all interactive elements
- Focus rings: `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`
- WCAG AA contrast: all color pairs verified (see `globals.css` audit table)
