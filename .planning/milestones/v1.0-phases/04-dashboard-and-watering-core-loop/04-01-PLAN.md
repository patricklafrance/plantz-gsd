## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Server Action | Untrusted form data crosses into server-side mutation |
| Session -> DB query | Session userId gates all Prisma queries for ownership |
| Cookie -> Server | user_tz cookie is untrusted client input used for date boundaries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | logWatering action | mitigate | Zod v4 safeParse validates all input fields; future-date refine blocks invalid wateredAt |
| T-04-02 | Elevation of privilege | logWatering action | mitigate | Ownership check: `db.plant.findFirst({ where: { id, userId, archivedAt: null } })` before mutation |
| T-04-03 | Elevation of privilege | editWateringLog action | mitigate | Ownership check through plant relation: `plant: { userId: session.user.id }` |
| T-04-04 | Elevation of privilege | deleteWateringLog action | mitigate | Same ownership check as T-04-03 |
| T-04-05 | Tampering | user_tz cookie | accept | Cookie only affects date display boundaries; worst case: plants shown in wrong urgency group for one session. No data corruption. |
| T-04-06 | Tampering | Demo user mutations | mitigate | `if (session.user.isDemo) return { error: "Demo mode..." }` in every Server Action |
| T-04-07 | Denial of service | Duplicate watering log | mitigate | Same-calendar-day check prevents log spam; WaterButton disabled during isWatering state |
