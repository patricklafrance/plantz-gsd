## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test input payload → Zod parse → guard | Tests construct schema-valid inputs so control flow reaches `requireHouseholdAccess`. Payload shape is trusted within the test scope; the goal is to reach the guard, not to validate Zod. |
| Mocked `requireHouseholdAccess` → action | The mock rejects with real ForbiddenError. Action's handling of the thrown error is what's being tested — does it propagate (preferred) or wrap (acceptable)? |
| ForbiddenError cross-module instanceof | `vi.importActual` ensures the test's ForbiddenError CLASS is the same class the action sees. Phase 1 `Object.setPrototypeOf` fix ensures `error instanceof ForbiddenError` is true even when thrown from a different module compile unit. |
