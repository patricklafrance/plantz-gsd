## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08-01 | Tampering | PlantCard href construction | mitigate | `householdSlug` is passed from Server Component (`await params`) which has already been validated by the layout's `getCurrentHousehold()` call — downstream components receive a slug that has already been membership-gated. Client cannot inject a different slug into the href without navigating to a different URL first, which would re-trigger the layout guard. |
| T-02-08-02 | Information Disclosure | Link targets | accept | Slugs are globally unique but not secret — they appear in the URL bar at all times. Exposing them in hrefs leaks no information the user didn't already have. |
| T-02-08-03 | Spoofing (open redirect) | Link construction using template literal | accept | Template literal `"/h/${householdSlug}/plants/${plant.id}"` is internal relative path only. Not a user-controlled redirect; cannot be weaponized for open-redirect. `plant.id` is server-trusted (came from Prisma query scoped to household). |
</threat_model>
