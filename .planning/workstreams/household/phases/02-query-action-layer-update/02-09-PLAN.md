<objective>
Fix UAT-9 (major): visiting `/h/{bogus-slug}/dashboard` returns HTTP 404 (correct) but renders a blank page in the browser. The `not-found.tsx` boundary at `src/app/(main)/h/[householdSlug]/not-found.tsx` exists and has valid JSX content (heading, icon, link) — yet the browser sees nothing. Something is preventing the `notFound()` throw from reaching that boundary.

**Root cause HYPOTHESIS (to be verified by Task 1, not assumed):** The current flow is `layout.tsx` → `getCurrentHousehold(slug)` → `notFound()`. Per Next.js 16 App Router docs (`.claude/skills/nextjs/references/error-handling.md`), `not-found.tsx` at segment X catches `notFound()` thrown from pages/layouts at segment X or deeper — same-segment throws ARE catchable in principle. However, when `notFound()` is thrown from the SAME segment's `layout.tsx` while that layout is itself rendering, the behavior depends on where in the render tree the throw happens: if the layout throws BEFORE it begins rendering children, the nearest boundary that can catch it may be the PARENT segment's `not-found.tsx` (which does not currently exist at `src/app/(main)/not-found.tsx`). Another possibility (Cause A0) is that the `not-found.tsx` IS wired correctly but another boundary is intercepting the render (e.g., a sibling `error.tsx` catching the `NEXT_NOT_FOUND` before the framework routes it, or the outer `(main)/layout.tsx` session gate returning empty markup).

Task 1 MUST distinguish among these causes with a real reproduction before Task 2 picks a fix.

Goal: ensure the bogus-slug URL renders a visible "Household not found" page in the browser. Must also preserve: (a) HTTP 404 status, (b) the layout's membership gate (user without session still gets redirected to /login by the outer layout), (c) the existing household-scoped `not-found.tsx` for unknown-plant-id cases.

Purpose: user-facing bug — a 404 on a bad slug looks like the app crashed. Trust and debuggability matter.

Output: diagnostic confirmation of the root cause, then a targeted fix (add a parent-segment boundary, inspect middleware, or isolate a not-found.tsx runtime error).
</objective>
