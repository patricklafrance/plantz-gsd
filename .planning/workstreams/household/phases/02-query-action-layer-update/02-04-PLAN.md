| householdId argument → Prisma where | Query functions accept `householdId` as an untrusted string parameter; Prisma's `findMany`/`findFirst` with `where: { householdId }` translates to SQL that filters by indexed column. Combined with the indexes from Phase 1 (`@@index([householdId, archivedAt])` on Plant, etc.), the filter is a direct SQL predicate with no injection surface (Prisma parameterizes). |
| Nested plant.householdId → JOIN | WateringLog/Note/Reminder queries JOIN to Plant on plantId (FK), then filter on plant.householdId. Two-step indexed lookup: plantId → Plant row, then plant.householdId match. Prisma generates the parameterized SQL. |
| Zod schema on querytime → NONE | Query functions typically don't Zod-parse; they rely on the caller (Server Component with `getCurrentHousehold` result) to supply a validated householdId. Plan 05 (actions) is where Zod parse happens. |
