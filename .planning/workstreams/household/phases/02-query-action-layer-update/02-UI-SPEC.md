# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Query + Action Layer Update. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Scope (read first)

Phase 2 is fundamentally a **data-layer and URL-routing migration**. CONTEXT.md D-07 and D-09 explicitly state NO visual UI is added for `createHousehold` or `getUserHouseholds` — those UIs ship in Phase 6. The household switcher, settings page, member list, invite forms, and rotation reorder are all Phase 6 territory and are **out of scope** for this contract.

The only UI-adjacent surfaces this phase touches are:

1. **New** `src/app/(main)/h/[householdSlug]/error.tsx` — 403 Forbidden state (user not a member of this household)
2. **New** `src/app/(main)/h/[householdSlug]/not-found.tsx` — 404 unknown household slug
3. **Relocation** `src/app/(main)/h/[householdSlug]/layout.tsx` adopts the chrome (header with logo + nav links, `NotificationBell`, `BottomTabBar`, demo banner) currently rendered by `src/app/(main)/layout.tsx` (per RESEARCH §Q11 Option A)
4. **Tab link update** `BottomTabBar` and outer nav `<Link href="/dashboard|/plants|/rooms">` targets rewritten to `/h/[slug]/dashboard|/plants|/rooms`; active-tab detection updated accordingly
5. **Legacy redirect stubs** at `src/app/(main)/{dashboard,plants,plants/[id],rooms,rooms/[id]}/page.tsx` — pure `redirect()` pages; render nothing visible except the transient fallback path described below

All existing page-level designs (dashboard, plants grid, plant detail, rooms, plant add dialog, log watering, etc.) are **unchanged** — file moves only.

**Out-of-scope flags (must not sneak in):** household switcher UI, household settings form, create-household form, member list, invite link UI, cycle banner, assignee-gating badges, default-household toggle. If any of these surface in planning, return to Phase 6 backlog.

## Copywriting Contract

Phase 2 introduces copy at exactly two new surfaces (`error.tsx`, `not-found.tsx`) plus one edge-case fallback for legacy redirect stubs. Tone matches Plant Minder's existing calm/supportive voice (see `src/components/shared/empty-state.tsx` consumers — e.g., `"Your collection is empty"` / `"Add a plant to begin building your collection."`).

| Element | Copy |
|---------|------|
| Primary CTA (this phase) | **Not applicable** — no new forms or actions with user-facing buttons. (`createHousehold` has no UI per D-07.) |
| Empty state heading | **Not applicable** — no new empty states. Existing `EmptyState` usages (plants page, dashboard) carry over unchanged. |
| Empty state body | **Not applicable** |
| Error state — 403 Forbidden (`error.tsx`) — heading | **"You don't have access to this household"** |
| Error state — 403 Forbidden — body | **"Your membership may have been removed or you may have followed an outdated link. Go back to your dashboard to see the households you belong to."** |
| Error state — 403 Forbidden — action label | **"Go to dashboard"** → `Link href="/dashboard"` (legacy stub will re-resolve to their active household) |
| Error state — generic (non-Forbidden throw caught by `error.tsx`) — heading | **"Something went wrong"** |
| Error state — generic — body | **"We couldn't load this household. Try again, or go back to your dashboard."** |
| Error state — generic — action label | **"Try again"** (calls `reset()` from Next.js error boundary props) |
| Not-found — 404 unknown slug (`not-found.tsx`) — heading | **"Household not found"** |
| Not-found — 404 unknown slug — body | **"This household doesn't exist, or it may have been deleted. Go back to see the households you're part of."** |
| Not-found — 404 — action label | **"Go to dashboard"** → `Link href="/dashboard"` |
| Legacy redirect stub transient fallback (user hits `/dashboard` with null `activeHouseholdId`) | **"Redirecting to your household..."** — body-sm text, centered, rendered inside the outer `(main)/layout.tsx` chrome; if `session.user.activeHouseholdId` is null the stub redirects to `/login` instead (no visible copy) |
| Destructive confirmation | **Not applicable** — no new destructive actions in this phase. Existing delete/archive dialogs (plants, rooms, notes) retain their current copy verbatim. |

**Tone rules (binding):**
- Use "household" (lowercase, noun) consistently; never "group", "team", or "workspace"
- Calm, direct, no exclamation marks, no emoji
- Every error/not-found surface MUST offer a recovery path (a link to `/dashboard`, or a `reset()` retry)
- No blame ("you provided a bad slug") — instead surface the situation neutrally ("this household doesn't exist")

## Visual Contract for New Surfaces

### `src/app/(main)/h/[householdSlug]/error.tsx`

Client Component (Next.js requires `"use client"` on error boundaries). Catches:
- `ForbiddenError` thrown by `requireHouseholdAccess` in the layout (D-03)
- Any other uncaught throw from child server components/server actions

**Structure:** reuse the existing `EmptyState` component pattern — do NOT introduce a new one.

```tsx
<EmptyState
  icon={ShieldAlert}          // or AlertCircle for generic error
  iconVariant="muted"         // muted container; icon itself is text-destructive for 403
  heading="You don't have access to this household"
  body="Your membership may have been removed or you may have followed an outdated link. Go back to your dashboard to see the households you belong to."
  action={
    <Link href="/dashboard">
      <Button variant="outline" size="sm">Go to dashboard</Button>
    </Link>
  }
/>
```

**Error discrimination:** inspect `error.name === "ForbiddenError"` to pick the 403 copy; any other name uses the generic copy with a `Button onClick={() => reset()}`.

**Layout:** renders inside the `(main)/layout.tsx` chrome (header + BottomTabBar remain visible). Full vertical centering via `EmptyState`'s existing `py-16` and parent page gutters.

### `src/app/(main)/h/[householdSlug]/not-found.tsx`

Server Component. Triggered by `notFound()` called from the layout when `resolveHouseholdBySlug` returns null.

```tsx
<EmptyState
  icon={SearchX}              // lucide: "search with an X" — matches "not found"
  iconVariant="muted"
  heading="Household not found"
  body="This household doesn't exist, or it may have been deleted. Go back to see the households you're part of."
  action={
    <Link href="/dashboard">
      <Button variant="outline" size="sm">Go to dashboard</Button>
    </Link>
  }
/>
```

**Layout:** renders inside the `(main)/layout.tsx` chrome.

### Legacy redirect stubs (`/dashboard/page.tsx`, `/plants/page.tsx`, etc.)

These are `async` Server Components whose body is essentially:

```tsx
const session = await auth();
if (!session?.user?.activeHouseholdId) redirect("/login");
const hh = await resolveHouseholdBySlugById(session.user.activeHouseholdId);
if (!hh) redirect("/login");
redirect(`/h/${hh.slug}/<suffix>`);
```

**Visual contract:** `redirect()` throws synchronously, so the component body never returns JSX in the common path. **No visible copy is rendered** under normal operation — the browser shows the target page immediately.

**Edge case — lookup is slow or fails unexpectedly:** If the DB lookup awaits for a perceptible time, Next.js renders the nearest `loading.tsx` (existing `src/app/(main)/dashboard/loading.tsx`, `plants/loading.tsx`, `rooms/loading.tsx` already exist and show the page-specific skeleton — they remain acceptable here). No new loading UI needed.

### Chrome relocation — `/h/[householdSlug]/layout.tsx`

The chrome currently in `src/app/(main)/layout.tsx` (demo banner, header with logo + Plants/Rooms nav, `UserMenu`, `NotificationBell`, `BottomTabBar`) **relocates** into the new household-scoped layout. Outer `(main)/layout.tsx` is reduced to: session check, `TimezoneSync`, `FocusHeading`, `SkipToContent` link — enough to support legacy redirect stubs.

**Zero visual redesign.** All JSX, Tailwind classes, icon choices, and accessibility attributes transfer verbatim. The chrome now has access to `getCurrentHousehold()` from `src/features/household/context.ts` (D-03) — `householdId` is passed into `getReminderCount` / `getReminderItems` per D-14.

**Link target updates (mechanical):**

| Component | Current `href` | New `href` |
|-----------|----------------|------------|
| Header logo link (`<Leaf />`) | `/dashboard` | `/h/${slug}/dashboard` |
| Header "Plants" nav link | `/plants` | `/h/${slug}/plants` |
| Header "Rooms" nav link | `/rooms` | `/h/${slug}/rooms` |
| `BottomTabBar` Dashboard tab | `/dashboard` | `/h/${slug}/dashboard` |
| `BottomTabBar` Plants tab | `/plants` | `/h/${slug}/plants` |
| `BottomTabBar` Rooms tab | `/rooms` | `/h/${slug}/rooms` |
| `BottomTabBar` reminder item click | `/plants/${plantId}` | `/h/${slug}/plants/${plantId}` |
| `NotificationBell` reminder item click | `/plants/${plantId}` | `/h/${slug}/plants/${plantId}` |

**Active-tab detection (existing logic in `BottomTabBar` lines 39-42) must adapt:** `exact` match on `pathname === "/h/${slug}/dashboard"` and prefix match on `pathname.startsWith("/h/${slug}/plants")` etc. `slug` becomes a prop on `BottomTabBar`, threaded from the household layout.

**Pass the slug down:** `BottomTabBar` signature grows a `householdSlug: string` prop; the TABS array is constructed inside the component using that slug. No visual change.

### BottomTabBar Alerts link (out of scope for Phase 2)

STATE.md flags a v1 tech-debt item: `BottomTabBar` Alerts "links to /dashboard instead of notifications" and `NotificationBell` is hidden on mobile. RESEARCH and CONTEXT both scope those fixes to **Phase 5** (Household Notifications). This phase **preserves the existing behavior verbatim** — do not fix the Alerts link target in Phase 2.

## Accessibility Contract

Phase 2 must not regress existing a11y posture. Confirmed checkpoints:

- **Skip-to-content link** (`<a href="#main-content">` in current outer layout) — preserved; moves with the chrome into the household layout
- **`FocusHeading`** component (existing, announces page title on route change) — preserved
- **Landmark roles:** `<header>`, `<main id="main-content">`, `<nav aria-label>` — all preserved in the relocation
- **Touch targets:** `min-h-[44px]` on all `BottomTabBar` buttons — preserved
- **Focus ring:** `focus-visible:ring-2 focus-visible:ring-ring/50` — preserved on tabs and bell
- **`aria-current="page"`** on active `BottomTabBar` tab — preserved and re-wired to new `/h/[slug]/...` pathname matching
- **New `error.tsx` and `not-found.tsx`** must set `role="alert"` on the container for screen-reader announcement OR rely on `FocusHeading` by using the same `<h2 tabIndex={-1}>` pattern used in `EmptyState` (the existing component already renders a semantic `<h2>`, which is sufficient — add `role="alert"` only if the error is the sole content of the page)

**403 Forbidden screen-reader copy:** "Forbidden. You don't have access to this household." — use an `aria-live="polite"` region on the error page container so users on screen readers get the status announcement immediately.

## Responsive Contract

Phase 2 introduces no new breakpoints. Confirmed from existing code:
