## What's NOT done

### gap-01 — BUG-01: timezone silent overwrite (Medium severity — gap-closure plan picked)

`Intl.supportedValuesOf("timeZone")` returns 418 IANA zones but does NOT include `"UTC"` or `"Etc/UTC"`. Households seeded with `timezone: "UTC"` (the default in `prisma/seed.ts`) silently fall back to `Africa/Abidjan` in the General form select. Any OWNER who saves the form overwrites the real timezone.

**File:** `src/components/household/settings/general-form.tsx` lines 68–80 (the `useMemo` that builds the timezones list).

**Fix (committed recommendation — the user agreed to fix now):**

1. In the `useMemo`, ensure the list always includes both `"UTC"` as a first-class option AND the current stored `household.timezone` if it's not already in the returned list. Simplest shape:
   ```ts
   const timezones = useMemo(() => {
     try {
       const zones = (Intl as typeof Intl & { supportedValuesOf?: (key: string) => string[] })
         .supportedValuesOf?.("timeZone") ?? [];
       const sorted = zones.length > 0 ? zones.slice().sort() : ["UTC"];
       // Ensure UTC is always present AND preserve any stored value not in the IANA list
       const withStored = new Set<string>(["UTC", ...sorted]);
       if (household.timezone) withStored.add(household.timezone);
       return Array.from(withStored).sort((a, b) => a === "UTC" ? -1 : b === "UTC" ? 1 : a.localeCompare(b));
     } catch {
       return ["UTC", household.timezone].filter(Boolean) as string[];
     }
   }, [household.timezone]);
   ```
2. Add a known-set check to `updateHouseholdSettingsSchema` in `src/features/household/schema.ts` so arbitrary strings can't sneak through the server action (defense in depth). Use the same `Intl.supportedValuesOf("timeZone") ∪ {"UTC"}` membership test, or hardcode a short allowlist — pick whichever your repo prefers.
3. Write / extend a test under `tests/phase-06/settings-general-form.test.tsx` that seeds `household.timezone = "UTC"` and asserts the select's selected option is `"UTC"` (not `"Africa/Abidjan"`).
4. Verify via the existing Chrome DevTools MCP flow: reseed, log in as `demo@plantminder.app`, visit `/h/tAn97yhW/settings`, confirm the select shows `UTC` selected; save without changing anything; DB `household.timezone` remains `"UTC"`.
