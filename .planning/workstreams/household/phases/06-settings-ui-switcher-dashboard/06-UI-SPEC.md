# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Settings UI + Switcher + Dashboard.
> Generated by gsd-ui-researcher. All values pre-populated from upstream artifacts (CONTEXT.md D-01..D-45, prior phase UI-SPECs, codebase scan). No user questions were required — CONTEXT.md was gathered in assumptions mode and user confirmed all assumptions.
>
> **Scope surfaces:** (1) `<HouseholdSwitcher>` in top-nav logo slot (desktop) and inside `UserMenu` (mobile); (2) `/h/[householdSlug]/settings/page.tsx` — five stacked card sections, role-branched; (3) `CycleCountdownBanner` — fifth dashboard banner for assignee steady-state; (4) availability two-picker date form within settings; (5) auth resolver + redirect stub sort-order change.

## Phase UI Scope (read first)

This phase **assembles** the owner-facing control surface and user-facing navigation layer on top of the data + action + notification foundation shipped by Phases 1–5. All Server Actions already exist; the three new ones (`setDefaultHousehold`, `updateHouseholdSettings`, `reorderRotation`) follow established patterns.

**Three connected surfaces, each with its own section below:**

1. **Household switcher** — `<HouseholdSwitcher>` client component in the top-nav logo slot (desktop) and inside `UserMenu` (mobile). Uses existing `dropdown-menu.tsx`. Navigates between households, preserving current route suffix (list routes) or falling back to the list root (detail routes).

2. **Settings page** — single page at `/h/[householdSlug]/settings/page.tsx`, Server Component, five stacked `<Card>` sections: General → Members + Rotation → Invitations → My Availability → Danger Zone. Role-branched: OWNERs see all five; MEMBERs see General (read-only), Members (read-only roster), My Availability (full CRUD), Danger Zone (leave only).

3. **Dashboard cycle-countdown banner** — new fifth banner `cycle-countdown-banner.tsx` for the current assignee when no unread cycle event is pending. Slots between `ReassignmentBanner` and `PassiveStatusBanner` per the Phase 5 D-13 render order.

**Explicitly NOT in this phase:** new schema migrations (all columns exist), changes to Phase 5's four banner components, notifications surface rework, demo seeding (Phase 7), cycle-snooze UI (Phase 8), `command` or `sheet` shadcn installs (deferred), any DnD library, any third-party date-range picker.

### Route-preservation logic (D-05)

```ts
function buildSwitchPath(currentPathname: string, newSlug: string): string {
  // currentPathname e.g. "/h/old-slug/plants/abc123"
  const segments = currentPathname.split("/"); // ["", "h", "old-slug", "plants", "abc123"]
  const suffix = segments.slice(3).join("/");   // "plants/abc123"

  // Detail route heuristics: has a 4th segment that looks like a CUID
  const detailPattern = /^[a-z0-9]{20,}$/i;
  const isDetailRoute = segments.length >= 5 && detailPattern.test(segments[4] ?? "");

  if (isDetailRoute) {
    return `/h/${newSlug}/${segments[3]}`; // fall back to list
  }
  return `/h/${newSlug}/${suffix}`;
}
```

Implemented as a utility in `src/components/household/household-switcher.tsx`. `onSelect` fires `router.push(buildSwitchPath(pathname, newSlug))`.

## Visual Contract: Cycle-Countdown Banner

### Render condition (D-25 mutual exclusivity)

```
if (!currentCycle || currentCycle.status !== 'active') return null
if (currentCycle.assignedUserId !== sessionUser.id) return null

const hasUnreadEvent = cycleEvents.some(
  e => e.readAt === null &&
       (e.type === 'cycle_started' || e.type.startsWith('cycle_reassigned_'))
);
if (hasUnreadEvent) return null; // CycleStart or Reassignment banner takes priority

render <CycleCountdownBanner ... />
```

### Dashboard insertion point (D-24)

Current banner order (Phase 5):
1. `FallbackBanner`
2. `CycleStartBanner` or `ReassignmentBanner` (mutually exclusive)
3. `PassiveStatusBanner`

After Phase 6:
1. `FallbackBanner`
2. `CycleStartBanner` or `ReassignmentBanner`
3. **`CycleCountdownBanner` ← NEW (replaces position between 2 and 3 for assignees)**
4. `PassiveStatusBanner`

`PassiveStatusBanner` is non-assignee-only (Phase 5 D-12 unchanged). So items 3 and 4 are mutually exclusive by role.

### Banner variants

**Normal variant (N > 1 day):**
```tsx
<div
  role="status"
  className="flex items-start gap-3 rounded-lg border border-accent/30 bg-accent/10 px-4 py-3"
>
  <Clock className="h-5 w-5 shrink-0 text-accent mt-0.5" aria-hidden />

    <p className="text-sm text-foreground">

        <> <span className="font-semibold">{nextAssigneeName}</span> is next.</>

    </p>

      Cycle ends {format(cycleEndDate, "EEE MMM d")}
    </p>
  </div>
</div>
```

**Urgency variant (N ≤ 1 day):**
Same shell, swap to `border-destructive/30 bg-destructive/10` and `<Clock className="... text-destructive">`. Copy change:
```
Last day — tomorrow passes to {nextAssigneeName}.
```
Or for single-member household:
```
Last day of this cycle — starts fresh tomorrow.
```

**Single-member household (any N):**
Suppress "X is next" line. Copy:
```
You're on rotation — {N} days left in this cycle.
```
No urgent-variant for single-member (no next assignee to signal urgency toward — executor discretion, recommend keeping the urgency variant on the last day for self-awareness).

### N calculation

```ts
const daysLeft = Math.max(0, differenceInDays(cycle.endDate, new Date()));
```

`date-fns` `differenceInDays` truncates toward zero; `Math.max(0, ...)` prevents negative display on overdue cycles that haven't yet been advanced by cron.

## Interaction Contract

### Rotation reorder (D-10, D-12)

Optimistic UI flow:
1. User clicks `ArrowUp` or `ArrowDown` on a member row.
2. Local `members` state updates immediately (swap the two items in the array).
3. `useTransition` wraps `reorderRotation({ householdId, orderedMemberUserIds: newOrder.map(m => m.userId) })`.
4. While pending: arrow buttons are `disabled` on ALL rows (prevents double-click race).
5. On success: `revalidatePath` from server action refreshes authoritative order.
6. On error: revert local state to pre-click order; `toast.error("Couldn't update rotation order. Try again.")`.

The disabled-all-during-pending rule prevents the user from clicking two arrows before the first action settles, which would send stale intermediate orders.

### Settings form submit (D-16)

RHF `handleSubmit` → `updateHouseholdSettings` Server Action via `useTransition`. No optimistic update on the form itself (fields remain at submitted values during pending; button disabled). On success: `toast.success("Household settings saved.")`. On RHF validation error: inline field-level error messages (shadcn `FormMessage`). On server error: `toast.error("Couldn't save settings. Try again.")`.

### Invite creation (D-20)

Dialog body replaces itself on action success (Phase A → Phase B). Implementation: local `React.useState<{ phase: "idle" | "success" | "error"; token?: string }>`. On form submit: set phase = "success" with the returned token; on error: set phase = "error". The Dialog does not close on success — user must click "Done" to dismiss. This ensures the copy-link affordance is clearly accessible.

### Copy to clipboard (D-39)

Uses `navigator.clipboard.writeText(url)`. On success: `toast.success("Link copied — share it with people you want to invite.")`. On failure (clipboard permission denied): `toast.error("Couldn't copy — try selecting and copying the link manually.")`.

### Toast consolidation (D-40)

All action toasts in Phase 6 use `sonner` (`src/components/ui/sonner.tsx`). No other notification library. Toast catalogue:

| Action | Toast type | Copy |
|--------|------------|------|
| Settings saved | success | `"Household settings saved."` |

| Rotation reordered | success | `"Rotation order updated."` (shown only if server confirms; if optimistic-only feels fast, suppress this) |

| Default household set | success | `"Default household updated."` |
| Default household failed | error | `"Couldn't update default. Try again."` |
| Invite created | — | (no toast; dialog transitions to Phase B in-place) |
| Link copied | success | `"Link copied — share it with people you want to invite."` |
| Clipboard denied | error | `"Couldn't copy — try selecting and copying the link manually."` |
| Invite revoked | success | `"Invite link revoked."` |
