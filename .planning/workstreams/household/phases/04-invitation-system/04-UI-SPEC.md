## Accessibility Contract

Phase 4 must not regress existing a11y posture and must meet these new-surface checkpoints:

- **`/join/[token]` page:** set `<main>` as the landmark; the minimal header is a `<Link>` not a `<header>` (no nav) â€” acceptable because there's no navigation to wrap. Add a single `<h1 className="sr-only">Plant Minder invitation</h1>` above the branch content so screen readers announce the page purpose regardless of which branch renders (`EmptyState` uses `<h2>` internally; `Card` uses `<h3>` via `CardTitle` â€” neither is the page-level heading).
- **Branch 5b Accept form:** the form has an `aria-describedby` pointing to the CardContent body paragraph so submitting via keyboard announces the household context. If the Server Action throws, `toast.error` renders with `role="status"` (sonner default) â€” acceptable per WCAG 4.1.3.
- **Destructive-leave dialog:** `ResponsiveDialog` wraps Radix Dialog (desktop) / Vaul Drawer (mobile) â€” both ship with `role="dialog"` + `aria-modal="true"` + focus trap + Escape-to-close built in. The `<AlertTriangle />` icon has `aria-hidden="true"`; the dialog title text carries the semantic meaning. Confirm button does NOT autofocus â€” focus lands on Cancel (shadcn Dialog default) to match destructive-confirm safety convention.
- **Icon-only elements:** all lucide icons in the branch renders have `aria-hidden="true"`. No icon-only buttons on Phase 4 surfaces (every button has a text label).
