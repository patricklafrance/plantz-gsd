    2. The current className `"text-base font-semibold truncate break-all"` becomes `"text-base font-semibold truncate"`.

    **In `src/components/plants/plant-card.tsx` (line 54):**
    1. Same fix: remove `break-all` from the nickname `<p>` className.
    2. The current className `"text-base font-semibold truncate break-all"` becomes `"text-base font-semibold truncate"`.

    **Why this fixes the height issue:** The `break-all` class (`word-break: break-all`) is overridden by `truncate` (`white-space: nowrap`), making it dead code. However, the presence of `break-all` can cause subtle layout differences in how the browser calculates intrinsic sizes during flex layout. Removing it ensures the text container behaves identically regardless of content length — `truncate` alone guarantees single-line rendering with ellipsis overflow, producing uniform content height.

    **Do NOT:**
    - Add `break-all` or `break-words` — `truncate` handles overflow.
    - Change the card structure, padding, or flex layout — only the nickname class needs fixing.
    - Add explicit min-height unless the above fix is insufficient (it should be sufficient since all three lines in the content area — nickname, species, room — render as single-line truncated text with identical line heights).
