## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon padding |
| sm | 8px | Compact element spacing, icon-only button internal gap |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card group gaps |
| xl | 32px | Layout gaps, page-level section breaks |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

**Exceptions:**

| Exception | Value | Reason |
|-----------|-------|--------|
| Minimum touch target | 44px | WCAG 2.5.5 / D-03 — all interactive elements |
| Bottom tab bar height | 56px (h-14) | Thumb-friendly fixed bottom nav; includes safe-area-inset-bottom padding |
| Bottom tab bar tab hit area | 44px minimum | Each tab item must meet 44px touch target |
| Main content bottom padding (mobile) | 64px (pb-16) | Prevents content hiding behind fixed bottom tab bar |
| iOS safe-area padding | env(safe-area-inset-bottom) | Bottom tab bar on iPhone with home bar |
| Focus ring offset | 4px (ring-offset-1) | Tight professional offset; multiple of 4; matches shadcn/ui `ring-ring/50` focus-visible pattern |

*Source: D-03 (touch targets), D-01 (bottom tab bar), 07-RESEARCH.md Pattern 2*

## Typography

Existing project font is Geist via `--font-sans`. Sizes and weights are pre-populated from the shadcn base-nova preset and confirmed by codebase audit.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px (text-2xl) | 600 (semibold) | 1.2 | Page `<h1>` — one per page, must have `tabIndex={-1}` |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | Section `<h2>` — urgency sections, dialog titles, empty state headings |
| Body | 16px (text-base) | 400 (regular) | 1.5 | Primary content, form labels, card text |
| Label | 14px (text-sm) | 400 (regular) | 1.5 | Supporting text, metadata, timestamps, badge labels |

**Weights used:** regular (400) and semibold (600) only.

**Heading hierarchy rule (D-11):** Each page has exactly one `<h1>`. Sections use `<h2>`. Subsections use `<h3>`. No skipped levels. The `<h1>` must include `tabIndex={-1}` to support programmatic focus after client-side navigation (D-12).

**Empty state headings:** Empty state headings are `<h2>` elements styled `text-xl font-semibold` (20px/600). Visual distinction from the page `<h1>` is achieved through layout context (indented within the content area, below the page title) — not through an intermediate font size. Using the same Heading size (20px) is intentional and keeps the type scale at 4 sizes.

*Source: shadcn base-nova preset defaults, 07-RESEARCH.md Pattern 5*

## Color

All values are OKLCH from `src/app/globals.css`. The design system uses semantic tokens, not raw hex.

| Role | Token | OKLCH Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | Page background, main surfaces |
| Secondary (30%) | `--card` / `--muted` | `oklch(1 0 0)` / `oklch(0.97 0 0)` | Cards, nav background, filter pills |
| Accent (10%) | `--accent` | `oklch(0.62 0.10 155)` (green) | Reserved list below |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | Destructive actions and overdue status only |

**Accent (`--accent`) is reserved for:**
- "Due today" status badge background (`bg-accent/15 text-accent`)
- "Log watering" button (WaterButton) active state
- Notification bell badge count
- Focus ring (`--ring` is set to `oklch(0.62 0.10 155)` — matches accent)
- "Save" / primary CTA on forms
- Active bottom tab bar icon

**Accent is NOT used for:**
- Overdue status (uses `--destructive`)
- General links (use `--foreground` or `--muted-foreground`)
- Upcoming / recently-watered badges (use `--muted` / `--outline`)

**Primary visual anchor:** The urgency section listing overdue and due-today plants is the primary visual anchor — it sits at the top of the main content area above all other sections. Its accent and destructive badge colors draw the eye immediately on load.

**Status indicator color + icon map (D-07 — no color-only status):**

| Status | Badge Variant | Icon | Icon Color | Text |
|--------|--------------|------|------------|------|
| Overdue | `destructive` | `AlertTriangle` (h-3 w-3, aria-hidden) | `text-destructive` | "Xd overdue" |
| Due today | custom: `bg-accent/15 text-accent border-accent/20` | `Droplets` (h-3 w-3, aria-hidden) | `text-accent` | "Due today" |
| Upcoming | `outline` | `Clock` (h-3 w-3, aria-hidden) | `text-muted-foreground` | "In Xd" |
| Recently watered | `secondary` | `CheckCircle2` (h-3 w-3, aria-hidden) | `text-muted-foreground` | "Watered Xh ago" |
| Not scheduled | `outline` | `HelpCircle` (h-3 w-3, aria-hidden) | `text-muted-foreground` | "Every Xd" |

**WCAG AA Contrast Audit Required (D-10):**
The following custom color usages must be verified at 4.5:1 for normal text during implementation:
- `text-accent` on `bg-accent/15` — HIGH RISK (semi-transparent bg lowers contrast)
- `text-destructive` on `bg-destructive/10` — HIGH RISK
- `text-muted-foreground` (`oklch(0.556 0 0)`) on `bg-background` — MEDIUM RISK
- Onboarding banner gradient text — verify if any banner has colored text

Use Chrome DevTools Accessibility panel (Computed > Color Contrast) for verification.

*Source: src/app/globals.css, 07-CONTEXT.md D-07/D-10, 07-RESEARCH.md Pattern 4/Pattern 9*

## Interaction Contracts

### Touch Targets (D-03)

All interactive elements must meet 44x44px minimum hit area. Visual size may be smaller; use padding expansion to increase hit area without altering layout:

```
-m-2 p-2  (adds 8px padding all sides = 16px larger, does not affect layout)
```

**Known violations requiring fix (from 07-RESEARCH.md Pattern 3):**

| Element | Current | Fix Strategy |
|---------|---------|-------------|
| `Button size="default"` | h-8 (32px) | Add `min-h-[44px]` on mobile-critical CTAs, or use `-my-1.5 py-1.5` |
| `Button size="icon"` | size-8 (32px) | Add `-m-1.5 p-1.5` to expand hit area |
| `Button size="icon-sm"` | size-7 (28px) | Add `-m-2 p-2` to reach 44px (28px + 8px×2 = 44px, meeting WCAG 2.5.5 minimum) |
| Filter chip pills | h-7 (28px) | Add `-my-2 py-2` expansion |
| Snooze pills (1d, 2d, 1w) | ~28px | Add `-my-2 py-2` expansion |
| UserMenu trigger | h-8 w-8 (32px) | Add `p-1.5` inside trigger |
| Back-to-catalog link | ~20px text | Wrap in `p-2` container |

`WaterButton` is already compliant at h-11 w-11 (44px). No change needed.

### Bottom Tab Bar (D-01)

- Visible ONLY below `sm:` breakpoint (640px). Use `sm:hidden` on the tab bar.
- Top header nav links (Plants, Rooms) hidden on mobile with `hidden sm:flex`.
- Notification bell stays in top header on desktop; moves to bottom tab bar (as 4th tab) on mobile.
- Fixed at `bottom-0`, `z-50`, full viewport width.
- Tab items: Dashboard (`LayoutDashboard`), Plants (`Leaf`), Rooms (`DoorOpen`), Notifications (`Bell`).
- Active tab: accent color icon + label. Inactive: `text-muted-foreground`.
- Bottom padding: `pb-[env(safe-area-inset-bottom)]` for iOS safe area.
- `aria-label="Main navigation"` on the `<nav>` element to distinguish from top nav.
- `<main>` wraps content with `pb-16 sm:pb-0` to prevent content hiding.

*Source: 07-CONTEXT.md D-01, 07-RESEARCH.md Pattern 1*

### Mobile Bottom Sheet (D-04)

Modals affected: Add Plant, Edit Plant, Log Watering (retroactive date), Edit Watering Log.

On mobile (`< sm:` breakpoint): use `@base-ui/react/drawer` with:
- `fixed bottom-0 left-0 right-0 rounded-t-xl`
- `max-h-[90vh] overflow-y-auto`
- Swipe-to-dismiss via Base UI Drawer's built-in `SwipeArea`
- `pb-[env(safe-area-inset-bottom)]` for iOS

On desktop (`sm:` and above): use existing `@base-ui/react/dialog` (centered modal).

Implementation: Create `src/components/ui/drawer.tsx` wrapping `@base-ui/react/drawer`. Create `src/components/shared/responsive-dialog.tsx` that switches based on `useMediaQuery("(max-width: 639px)")`.

*Source: 07-CONTEXT.md D-04, 07-RESEARCH.md Pattern 2*

### Keyboard Navigation (D-05)

- Tab order: Skip-to-content link → top header → main content → (on mobile) bottom tab bar.
- Skip-to-content link: visually hidden (`sr-only`), appears on focus (`focus:not-sr-only`), positioned `absolute top-4 left-4 z-50`, styled with `bg-background px-4 py-2 text-sm font-medium rounded-md ring-2 ring-ring`.
- `<main>` receives `id="main-content"` as the skip link target.
- Escape closes dialogs and drawers (Base UI handles this).
- Enter/Space activates buttons and links (Base UI handles this).
- Focus rings: `focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50` — already on all Base UI primitives. Do NOT add `outline: none` without a visible replacement.

*Source: 07-CONTEXT.md D-05, 07-RESEARCH.md Pattern 6*

### Focus After Navigation (D-12)

After every client-side route change, move focus to the page `<h1>`.

- Hook: `src/hooks/use-focus-heading.ts` — uses `usePathname` + `useEffect` to call `document.querySelector("h1")?.focus()`.
- Wrapper: `src/components/shared/focus-heading.tsx` — thin Client Component, placed in `(main)/layout.tsx`.
- Every page `<h1>` must have `tabIndex={-1}` and `className="... outline-none"` (suppress focus ring on programmatic focus — ring only needed for keyboard tab focus).

*Source: 07-CONTEXT.md D-12, 07-RESEARCH.md Pattern 5*

### ARIA Live Regions (D-06)

Live regions are triggered ONLY for these key mutations (via existing Sonner `toast()` calls):
- Watering logged
- Watering log failed (+ retry action)
- Reminder snoozed
- Plant added
- Plant archived / deleted
- Form validation errors (already handled by shadcn Form `aria-invalid`)

Filter changes, sort changes, and search input do NOT trigger live regions.

Watering failure toast must include retry action:
```
toast.error("Couldn't log watering.", { action: { label: "Retry", onClick: () => handleWater(plant) } })
```

*Source: 07-CONTEXT.md D-06, 07-RESEARCH.md Pattern 8*

### Landmark Roles (D-13)

Each layout group must use correct semantic elements:

| Element | `aria-label` | Notes |
|---------|-------------|-------|
| `<header>` | (none required — only one header) | Top app header |
| `<nav>` (top) | `"Top navigation"` | Houses dashboard/plants/rooms links on desktop |
| `<nav>` (bottom tab bar) | `"Main navigation"` | Mobile bottom tab bar — must differ from top nav label |
| `<main id="main-content">` | (none required) | Primary content area, skip link target |
| `<footer>` | (none required) | If present |

*Source: 07-CONTEXT.md D-13*

### Pagination (D-15)

Plants collection page only. Dashboard sections are NOT paginated.

- Page size: 20 plants per page.
- URL param: `page` (integer, default 1). Follows same pattern as existing `room`, `search`, `status`, `sort` params.
- UI: "Previous" and "Next" buttons as `<Link>` components preserving all current query params. Show current page and total pages (`Page 2 of 5`).
- No infinite scroll. No page-number button bar (Previous/Next only for simplicity).
- Total count and pagination must use `Promise.all([getPlants(...), db.plant.count({ where })])` to avoid serial queries.

*Source: 07-CONTEXT.md D-15, 07-RESEARCH.md Pattern 7*

### Character Limits (D-14)

| Field | Max Characters | Input Attribute |
|-------|---------------|----------------|
| Plant nickname | 40 | `maxLength={40}` |
| Room name | 40 | `maxLength={40}` |
| Note content | 1000 | `maxLength={1000}` |

Display remaining character count when the user is within 20 characters of the limit (e.g., "23/40").

*Source: 07-CONTEXT.md D-14 (40 char limit chosen from the "e.g." in context)*

### Error States (D-16)

| Error Type | UI Pattern |
|------------|-----------|
| Server Action failure | Inline error message below the affected field OR toast if no field context |
| Watering log failure | `toast.error` with Retry action button |
| Network timeout (loading) | Skeleton components (`src/components/ui/skeleton.tsx`) — already installed |
| No data (empty state) | EmptyState component (see Copywriting Contract below) |
| Pagination out of range | Redirect to page 1 of results |

*Source: 07-CONTEXT.md D-16*

## Accessibility Checklist

This section defines what "done" means for WCAG AA compliance in this phase.

| Requirement | Contract |
|-------------|---------|
| Color contrast | All text on custom color backgrounds passes 4.5:1 for normal text, 3:1 for large text. Verified in Chrome DevTools. |
| Non-color status | Every status badge has icon + text label + color. No color-only status. |
| Keyboard navigation | Tab order is logical. All interactive elements reachable. No keyboard traps except modals (Base UI handles). |
| Focus rings | Visible on all interactive elements in `focus-visible` state. `focus-visible:ring-3 focus-visible:ring-ring/50` present on all Base UI components. |
| Skip link | First tabbable element on every page. Links to `#main-content`. Visually hidden until focused. |
| Landmark roles | `<header>`, `<nav>` (x2 with distinct aria-labels), `<main id="main-content">` present in all layouts. |
| Heading hierarchy | One `<h1>` per page. `<h2>` for sections. `<h3>` for subsections. No skipped levels. |
| Form labels | All form inputs have associated `<label>` elements (shadcn Form handles this). |
| ARIA live | Key mutations use Sonner toast (polite live region). No live regions on filter/sort/search. |
| Screen reader | All images/icons that are decorative have `aria-hidden="true"`. All icons with meaning have visible text labels. |
| Focus after navigation | After route change, focus moves to page `<h1>` via FocusHeading component. |
| Touch targets | All interactive elements have 44x44px minimum hit area. |
| Mobile dialogs | Modals become bottom sheet drawers on `< sm:` breakpoint. |

*Source: REQUIREMENTS.md UIAX-01–UIAX-04, 07-CONTEXT.md D-05 through D-13*
