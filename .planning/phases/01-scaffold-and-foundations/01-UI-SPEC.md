# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase 1 scope note:** This phase delivers zero user-facing screens. The UI contract here defines the foundational design system — design tokens, global CSS, root layout shell, and shadcn/ui initialization — that all subsequent phases (2-7) inherit. The executor's deliverable is a working token system and component scaffold, not rendered product pages.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Compact element spacing, button inner padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form field gaps |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks, page hero padding |
| 3xl | 64px | Page-level vertical spacing |

Exceptions: Touch targets on interactive elements must be a minimum of 44px in height/width (WCAG 2.5.5 AA). This is enforced via `min-h-[44px]` utilities on tap targets, not via the spacing scale.

## Typography

Font family: **Geist** for all text (sans-serif). Loaded via `next/font/google` or the built-in Next.js 16 Geist bundle.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Paragraphs, descriptions, form labels |
| Label | 14px | 500 (medium) | 1.4 | Nav items, badges, table headers, input labels |
| Heading | 20px | 600 (semibold) | 1.2 | Section headings, card titles, page sub-headings |
| Display | 28px | 600 (semibold) | 1.2 | Page titles, onboarding headlines |

Weights in use: **400** (regular) and **600** (semibold). Weight 500 is permitted for Label role only as a visual midpoint for UI chrome; it does not count as a third weight for body or heading content. If a single-weight constraint is required, collapse Label to 600.

## Color

Color system: **OKLCH** (Tailwind v4 default with shadcn/ui). Defined entirely in CSS `@theme` block — no Tailwind config file.

| Role | OKLCH Value | Hex Approx | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `oklch(97% 0.004 286)` | ~`#F4F4F5` Zinc-50 | Page background, main content surface |
| Secondary (30%) | `oklch(94% 0.006 286)` | ~`#E4E4E7` Zinc-100 | Cards, sidebar, nav background, input backgrounds |
| Accent (10%) | `oklch(62% 0.10 155)` | ~`#5C8A6A` Sage green | Reserved-for list below |
| Destructive | `oklch(55% 0.22 27)` | ~`#DC2626` Red-600 | Destructive actions only |

**Accent reserved for (explicit list):**
- Primary action buttons ("Add Plant", "Log Watering", "Save")
- Active nav item indicator / selected state
- Focus ring on interactive elements (2px, `ring-offset-2`)
- Watering status badge for "Due Today"
- Progress/completion indicators

**Accent is NOT used on:**
- Headings or body text
- Card borders (use Zinc-200)
- All-overdue status (use Destructive)
- Recently Watered status (use Zinc-400 muted)

**Dark mode:** Tokens will be defined with CSS custom properties supporting a `.dark` class toggle. Phase 1 defines both light and dark token values even though no UI renders yet. Dark surface values: Dominant → `oklch(14% 0.006 286)` (Zinc-950), Secondary → `oklch(20% 0.006 286)` (Zinc-900).

## Token Implementation Notes

These are executor instructions for wiring the design contract into the codebase.

### Global CSS (`src/app/globals.css`)

Define all tokens in the `@theme` block using CSS custom properties:

```css
@import "tailwindcss";

@theme {
  /* Font */
  --font-sans: "Geist", ui-sans-serif, system-ui, sans-serif;

  /* Spacing (Tailwind v4 extends default scale — these are aliases) */
  --spacing-xs: 4px;
  --spacing-sm: 8px;
  --spacing-md: 16px;
  --spacing-lg: 24px;
  --spacing-xl: 32px;
  --spacing-2xl: 48px;
  --spacing-3xl: 64px;

  /* Colors — light mode */
  --color-background: oklch(97% 0.004 286);
  --color-surface: oklch(94% 0.006 286);
  --color-accent: oklch(62% 0.10 155);
  --color-accent-foreground: oklch(98% 0.004 155);
  --color-destructive: oklch(55% 0.22 27);
  --color-destructive-foreground: oklch(98% 0.004 27);
  --color-foreground: oklch(14% 0.006 286);
  --color-muted: oklch(60% 0.006 286);
  --color-border: oklch(88% 0.006 286);
  --color-ring: oklch(62% 0.10 155);
}

@layer base {
  .dark {
    --color-background: oklch(14% 0.006 286);
    --color-surface: oklch(20% 0.006 286);
    --color-foreground: oklch(97% 0.004 286);
    --color-muted: oklch(60% 0.006 286);
    --color-border: oklch(28% 0.006 286);
  }
}
```

### Root Layout (`app/layout.tsx`)

- Apply `font-sans` to `<html>` or `<body>`
- Set `<title>` and `<meta name="description">` from the copywriting contract above
- Include a `ThemeProvider` wrapper (client component) that reads a `theme` cookie for SSR-safe dark mode
- No nav, sidebar, or page-level chrome in root layout — route group layouts own that

### `cn()` Utility (`src/lib/utils.ts`)

shadcn/ui auto-generates this. Confirm it exists after `npx shadcn@latest init`:

```ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```
