## Spacing Scale

Declared values from `globals.css` `@theme` — multiples of 4px:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Between status badge and label, kebab menu items |
| md | 16px | Card internal padding, dialog content padding |
| lg | 24px | Section vertical gap, page-level padding |
| xl | 32px | Between dashboard urgency sections |
| 2xl | 48px | Empty state icon top/bottom breathing room |
| 3xl | 64px | Empty state full vertical centering offset |

Exceptions:
- Water button touch target: minimum 44x44px (WCAG 2.5.5) — use `h-11 w-11` (44px) for the icon button, even though icon itself is 20px
- History list row height: minimum 44px for mobile tap target

## Typography

Established in prior phases. All sizes in px as rendered by Tailwind v4 defaults:

| Role | Size | Weight | Line Height | Tailwind Class | Source |
|------|------|--------|-------------|----------------|--------|
| Body | 16px | 400 (regular) | 1.5 | `text-base` | globals.css base styles |
| Label / caption | 14px | 400 (regular) | 1.5 | `text-sm` | plant-card.tsx, plant-detail.tsx |
| Micro / metadata | 12px | 400 (regular) | 1.4 | `text-xs` | plant-card.tsx room name |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` | dashboard/page.tsx h1 |

Rules:
- Only 2 weights used: 400 (regular) and 600 (semibold). No other weights.
- Muted foreground (`text-muted-foreground`) for secondary/metadata text.
- Truncation (`truncate`) on plant nickname and species in list contexts.

## Color

All values are OKLCH CSS variables from `globals.css`. Hex approximations shown for human reference only — use CSS variables in code.

| Role | CSS Variable | OKLCH | Approx Hex | Usage |
|------|-------------|-------|------------|-------|
| Dominant (60%) | `--color-background` | `oklch(97% 0.004 286)` | #F5F5F7 | Page background, empty areas |
| Secondary (30%) | `--color-surface` / `--card` | `oklch(94% 0.006 286)` / `oklch(1 0 0)` | #F0F0F3 / #FFF | Cards, sidebar, nav, card backgrounds |
| Accent (10%) | `--color-accent` / `--accent` | `oklch(62% 0.10 155)` | ~#3D9E6E | See accent reserved-for list below |
| Destructive | `--color-destructive` | `oklch(55% 0.22 27)` | ~#D94F2B | Delete watering log, destructive confirm button |
| Foreground | `--color-foreground` | `oklch(14% 0.006 286)` | ~#1A1A1F | Body text, headings |
| Muted | `--color-muted` | `oklch(60% 0.006 286)` | ~#888890 | Secondary text, metadata, placeholders |
| Border | `--color-border` | `oklch(88% 0.006 286)` | ~#DDDDE2 | Card borders, dividers, input outlines |

**Accent reserved for (explicit list — not general interactive elements):**
1. Water droplet icon button (the one-tap water action) — fill `bg-accent`, icon `text-accent-foreground`
2. Urgency badge: "Due today" — `bg-accent/15 text-accent` (subtle tint, not full fill)
3. Plant icon avatar background — `bg-accent/10` with `text-accent` icon (existing pattern)
4. Focus ring on interactive elements — `outline-ring/50` (existing global rule)
5. Active/selected state on dashboard section headers (if section is filtered)

**Second semantic color — status urgency palette (Phase 4 addition):**

These are inline tints applied only to status badges, not CSS variable additions:

| Status | Badge Approach | Tailwind classes |
|--------|---------------|-----------------|
| Overdue | Red tint | `bg-destructive/10 text-destructive border-destructive/20` |
| Due Today | Accent tint | `bg-accent/15 text-accent border-accent/25` |
| Upcoming | Muted | `variant="outline"` (existing Badge default) |
| Recently Watered | Muted green tint | `bg-accent/8 text-muted-foreground border-border` |

**Status must not rely on color alone.** Each badge also carries text ("Overdue", "Due today", "In 2d", "Watered") per UIAX-03.

## Interaction Contracts

### 1. One-Tap Water Button (UIAX-05 — Optimistic UI)

**Element:** Droplet icon button on every dashboard plant card, right side.

**Primary visual anchor:** The urgency status badge is the primary visual anchor on each card — it communicates the action needed at a glance. The water droplet button is the primary interactive focal point, positioned on the right so the eye travels from status to action.

**Anatomy:**
```
[PlantCard] [Leaf icon | Nickname / species / room | Status badge | [Droplet button]]
```

**Button spec:**
- Icon: `Droplet` from lucide-react, size `h-5 w-5`
- Container: `h-11 w-11` rounded-full, `bg-accent/10 hover:bg-accent/20`
- Color: `text-accent` default, `text-accent-foreground bg-accent` on hover
- Touch target: 44x44px (enforced by h-11 w-11)
- `aria-label="Water [plant nickname]"` — required for screen reader

**Optimistic flow (D-06):**
1. User taps — button shows loading spinner immediately (replace Droplet icon with `Loader2` spinning, `animate-spin`)
2. Card fades out from its current section: `transition-all duration-300 opacity-0 scale-95`
3. Toast appears: "Monstera watered! Next: Apr 17" (use plant nickname, format date as "MMM d")
4. On server confirmation: card is removed from DOM and reappears in Recently Watered on next server render
5. On server error: card fades back in, toast changes to error state: "Couldn't log watering. Try again."

**Duplicate debounce (D-11):**
- Server Action returns `{ error: "DUPLICATE" }` for same plant within 60 seconds
- Toast: "Already logged! Edit from history if needed."
- No card animation — card stays in place

### 2. Dashboard Urgency Sections (D-01, D-02, D-03)

**Section structure:**
```
[Section header: "Overdue (3)"] ← h2, text-base font-semibold
[Plant card grid]                ← 1-col mobile, 2-col md:, same as plant collection
[Section header: "Due Today (1)"]
...
```

**Section header spec:**
- `text-base font-semibold` for section label
- Count in parentheses: `text-muted-foreground font-normal` — same line, not a badge
- No visual divider between header and grid — `gap-sm` spacing only
- Separator (`<Separator />`) between sections if more than one section renders

**Hidden when empty (D-02):** Section does not render at all if count is 0. No empty section placeholder.

**Sort order within sections (D-03):**
- Overdue: most days late first (most negative daysUntil)
- Due Today: alphabetical by nickname
- Upcoming: soonest due first (ascending daysUntil)
- Recently Watered: most recently watered first (descending wateredAt)

**Card grid:**
- Mobile (< 768px): 1-column, full width
- Desktop (≥ 768px): 2-column grid, `grid-cols-1 md:grid-cols-2 gap-md`

### 3. Status Badge Display (DASH-01)

Badge text (not just color) conveys urgency — required by UIAX-03:

| Condition | Badge text | Example |
|-----------|-----------|---------|
| daysOverdue ≥ 1 | "{N}d overdue" | "3d overdue" |
| daysOverdue = 0 | "Due today" | "Due today" |
| daysUntil 1–7 | "In {N}d" | "In 2d" |
| daysUntil > 7 | "In {N}d" | "In 12d" |
| Recently watered | "Watered {N}d ago" | "Watered 1d ago" |

### 4. Log Watering Dialog (D-09)

**Trigger:** "Log watering" button on plant detail page.

**Dialog spec:**
- Title: "Log watering"
- Date field: shadcn Calendar in a Popover, defaults to today, allows any past date, no future dates
- Optional note field: `<Input>` with `placeholder="Optional note (e.g. used filtered water)"`, maxLength 280
- Submit CTA: "Log watering" (primary button, `bg-accent text-accent-foreground`)
- Dismiss: "Don't log" (ghost button)
- On success: dialog closes, sonner toast: "[Nickname] watered on [date]. Next: [date]"

**Date picker display format:** "April 14, 2026" (full month name, date-fns `format(date, "MMMM d, yyyy")`)
**Date picker calendar format:** Week starts Monday.

### 5. Watering History List (D-08, D-10)

**Location:** Plant detail page, within existing `<Card>` "Watering history" section.

**Entry anatomy (most recent first):**
```
[Date (formatted)] · [relative time]      [Kebab ⋮]
[optional note text — text-sm text-muted-foreground]
```

**Date format:** `format(wateredAt, "MMMM d, yyyy")` — e.g. "April 11, 2026"
**Relative time:** `"3 days ago"` / `"Today"` / `"Yesterday"` — derived with date-fns `formatDistanceToNow`
**Kebab menu items:** "Edit", "Delete" — using shadcn DropdownMenu
**Row min-height:** 44px for mobile tap target
**Pagination threshold:** Show latest 20 entries. If > 20 exist, show "Load more" text button (not a full pagination control). Load next 20 on click.

### 6. Edit Watering Log Dialog

**Trigger:** Kebab "Edit" on a history entry.

**Dialog spec:**
- Title: "Edit watering log"
- Date field: same Calendar/Popover as log dialog, pre-filled with entry's `wateredAt`
- Note field: pre-filled with existing note
- Submit CTA: "Save changes"
- Dismiss: "Discard changes" (ghost button)
- On success: entry updates in place, toast: "Watering log updated."

### 7. Delete Watering Log (D-10)

**Trigger:** Kebab "Delete" on a history entry.

**Pattern:** shadcn `<AlertDialog>` (matches Phase 3 delete plant pattern).

**AlertDialog spec:**
- Title: "Delete watering log?"
- Body: "This will remove the log from [Nickname]'s history and recalculate the next watering date."
- Destructive CTA: "Delete log" — `variant="destructive"`
- Dismiss: "Keep log" (ghost button)

### 8. Loading Skeletons (DASH-04)

Dashboard uses skeleton placeholders during initial server render on slow connections. Implemented as Suspense fallback:

**Skeleton structure per card:**
```
[Skeleton h-10 w-10 rounded-md] [Skeleton h-4 w-32] [Skeleton h-5 w-16]
```
Repeat 3-4 times per section. Use existing `<Skeleton />` component.

**Section header skeleton:** `<Skeleton className="h-5 w-24" />` above each card group.

### 9. Dashboard Empty State (All Plants Recently Watered)

When user has plants but none need attention (all recently watered, none overdue or due):

**Empty state spec:**
- Icon: `CheckCircle2` from lucide-react, `h-8 w-8 text-accent`
- Container: same rounded-full accent/10 background as existing empty state
- Heading: "All caught up!"
- Body: "Your plants are all watered. Check back when the next one is due."
- No CTA (nothing to do)

**When user has no plants at all (existing empty state — preserve):**
- Heading: "No plants yet"
- Body: "Add your first plant to start tracking your watering schedule."
- CTA: "Add plant" — `<AddPlantDialog />`

## Animation Contract

All animations use CSS transitions via Tailwind — no external animation library.

| Interaction | Animation | Duration | Easing |
|-------------|-----------|----------|--------|
| Water button tap (optimistic) | `opacity-0 scale-95` on card | 300ms | `ease-out` |
| Card fade-in to Recently Watered | `opacity-100 scale-100` from `opacity-0` | 200ms | `ease-in` |
| Loading spinner on water button | `animate-spin` (Tailwind built-in) | — | — |
| Dialog open/close | shadcn default (Radix) | — | — |
| Toast appear | Sonner default | — | — |
| Skeleton shimmer | `tw-animate-css` pulse (already imported) | — | — |

**Reduce motion:** All card animations must respect `prefers-reduced-motion`. Use `motion-safe:` Tailwind prefix on transition/animation classes.
