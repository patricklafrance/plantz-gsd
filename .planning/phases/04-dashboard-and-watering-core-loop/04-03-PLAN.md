    A client component (`"use client"`) that serves both "log new" and "edit existing" watering modes. Follow the dialog pattern established by `add-plant-dialog.tsx` and `edit-plant-dialog.tsx`.

    Props:
    ```typescript
    interface LogWateringDialogProps {
      plantId: string;
      plantNickname: string;
      // Edit mode props (optional — when provided, dialog is in edit mode)
      editLog?: { id: string; wateredAt: Date; note: string | null };
      open?: boolean;
      onOpenChange?: (open: boolean) => void;
    }
    ```

    When `editLog` is undefined: "Log watering" mode (D-09). When `editLog` is provided: "Edit watering log" mode.

    Form implementation:
    - Use `react-hook-form` with `@hookform/resolvers/zod` and the appropriate schema (`logWateringSchema` for new, `editWateringLogSchema` for edit)
    - Date field: shadcn Calendar inside a Popover. Default: today (new mode) or `editLog.wateredAt` (edit mode). Max date: today (no future). The Popover trigger is a Button showing the formatted date (`format(date, "MMMM d, yyyy")`) or "Pick a date" if none selected.
    - Note field: `<Input placeholder="Optional note (e.g. used filtered water)" maxLength={280} />` per UI-SPEC copywriting contract
    - Calendar: `<Calendar mode="single" selected={date} onSelect={setDate} disabled={(date) => date > new Date()} weekStartsOn={1} />` — week starts Monday per UI-SPEC

    Submit handling:
    - New mode: `const result = await logWatering({ plantId, wateredAt: date, note: noteValue || undefined })`. On success: close dialog, `toast(plantNickname + " watered on " + format(date, "MMMM d") + ". Next: " + format(new Date(result.nextWateringAt), "MMMM d"))`. On DUPLICATE error: `toast("Already logged! Edit from history if needed.")`. On other error: `toast.error("Couldn't log watering. Try again.")`
    - Edit mode: `const result = await editWateringLog({ logId: editLog.id, wateredAt: date, note: noteValue || undefined })`. On success: close dialog, `toast("Watering log updated.")`. On error: `toast.error(result.error)`

    - Edit mode title: "Edit watering log", submit CTA: "Save changes", dismiss: "Discard changes" (`variant="ghost"`)
    - Both: loading state on submit button with `Loader2 animate-spin` while submitting

    Standalone trigger (new mode): A `<Button>` with text "Log watering" that opens the dialog. When `open`/`onOpenChange` props are not provided, the dialog manages its own open state (uncontrolled mode). When provided (edit mode from kebab), parent controls state.

| T-04-15 | Tampering | loadMoreHistory — read another user's logs | mitigate | Server Action verifies auth + ownership before returning data |
| T-04-16 | Tampering | Note field XSS | accept | React JSX auto-escapes; no `dangerouslySetInnerHTML`; Zod max 280 chars |
</threat_model>
