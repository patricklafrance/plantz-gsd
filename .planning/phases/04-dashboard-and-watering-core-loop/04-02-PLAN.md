## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser -> Server Component | Cookie-based timezone input is untrusted |
| Client optimistic state -> Server Action | Optimistic UI shows unconfirmed state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Spoofing | user_tz cookie | accept | Cookie only affects display date boundaries; no data mutation. Worst case: wrong urgency group shown for one session. TimezoneWarning component alerts on mismatch. |
| T-04-09 | Information disclosure | Dashboard page | mitigate | Auth check at top of DashboardPage: `if (!session?.user?.id) redirect("/login")`. userId gates all Prisma queries. |
| T-04-10 | Repudiation | Optimistic UI rollback | accept | useOptimistic automatically rolls back on Server Action failure; user sees error toast with retry option. No permanent state change without server confirmation. |
</threat_model>
