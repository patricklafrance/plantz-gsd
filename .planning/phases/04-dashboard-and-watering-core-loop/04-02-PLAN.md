    **Step 2: Create `src/components/watering/dashboard-client.tsx`**

    This is the main client wrapper that manages optimistic state for the entire dashboard. It is a `"use client"` component.

    Props: `{ groups: GroupedDashboardPlants }` — receives the pre-sorted, pre-grouped data from the Server Component.

    Implementation using React 19 `useOptimistic`:

    ```typescript
    "use client";
    import { useOptimistic, startTransition } from "react";
    import { toast } from "sonner";
    import { format } from "date-fns";
    import { logWatering } from "@/features/watering/actions";
    import { DashboardSection } from "./dashboard-section";
    import { DashboardPlantCard } from "./dashboard-plant-card";
    import { Separator } from "@/components/ui/separator";
    import type { DashboardPlant } from "@/types/plants";

    type GroupedDashboardPlants = {
      overdue: DashboardPlant[];
      dueToday: DashboardPlant[];
      upcoming: DashboardPlant[];
      recentlyWatered: DashboardPlant[];
    };
    ```

    State management:
    - `const [optimisticGroups, removeFromGroups] = useOptimistic(groups, (current, plantId: string) => { ... })` — the reducer filters the plantId out of ALL groups (overdue, dueToday, upcoming, recentlyWatered). This causes the card to disappear optimistically.
    - `const [wateringPlantIds, setWateringPlantIds] = useState<Set<string>>(new Set())` — tracks which plants are currently being watered (for loading spinner on button)

    `handleWater(plant: DashboardPlant)` function:
    1. Add plantId to `wateringPlantIds` set
    2. Call `startTransition(async () => { ... })`:
       a. `removeFromGroups(plant.id)` — card disappears optimistically
       b. `const result = await logWatering({ plantId: plant.id })`
       c. Remove plantId from `wateringPlantIds`
       d. If `result.error === "DUPLICATE"`: `toast("Already logged! Edit from history if needed.")`
       e. Else if `result.error`: `toast.error("Couldn't log watering. Try again.")`
       f. Else (success): `toast(plant.nickname + " watered! Next: " + format(new Date(result.nextWateringAt), "MMM d"))` — per D-06 copywriting contract

    Render structure:
    ```tsx
    const sections = [
      { key: "overdue", title: "Overdue", plants: optimisticGroups.overdue },
      { key: "dueToday", title: "Due Today", plants: optimisticGroups.dueToday },
      { key: "upcoming", title: "Upcoming", plants: optimisticGroups.upcoming },
      { key: "recentlyWatered", title: "Recently Watered", plants: optimisticGroups.recentlyWatered },
    ].filter(s => s.plants.length > 0);

    return (
      <div className="space-y-xl">
        {sections.map((section, index) => (
          <DashboardSection
            key={section.key}
            title={section.title}
            plants={section.plants}
            showSeparator={index > 0}
            renderCard={(plant) => (
              <DashboardPlantCard
                key={plant.id}
                plant={plant}
                onWater={() => handleWater(plant)}
                isWatering={wateringPlantIds.has(plant.id)}
                isRemoving={/* check if plant was optimistically removed — this is handled by it not being in the list */}
              />
            )}
          />
        ))}
      </div>
    );
    ```

    Note: The `isRemoving` animation is triggered by the card being about to be removed from the list. The simplest approach: use `useOptimistic` to immediately filter the plant out — the card simply disappears. For the fade-out animation per D-06, use a local state approach: when `handleWater` is called, add the plantId to a "removing" set, wait 300ms (animation duration), THEN call the optimistic remove. Or use CSS `transition` on the card with a `data-removing` attribute.

    Recommended animation approach: Add plantId to a `removingIds` Set state. The DashboardPlantCard reads `isRemoving={removingIds.has(plant.id)}` and applies `opacity-0 scale-95` classes. After 300ms (matching the CSS transition), the optimistic reducer removes the plant from the list. Use `setTimeout(() => removeFromGroups(plant.id), 300)` inside the startTransition.
