**Status filter pill color palette (Phase 5 addition):**

| Status Filter | Inactive pill | Active pill | Approach |
|---------------|--------------|-------------|----------|
| All | `variant="outline"` | `variant="default"` | Existing room filter pattern |

## Interaction Contracts

### 1. Search Bar (SRCH-01, D-07, D-08)

**Location:** `/plants` collection page, above the room filter pill row.

**Anatomy:**
```
[Search icon (Search, h-4 w-4, text-muted-foreground)] [Input — "Search plants..."] [Clear button × — only when value non-empty]
```

**Spec:**
- Use shadcn `<Input>` with a wrapping `<div className="relative">` for icon positioning
- Search icon: `Search` from lucide-react, `absolute left-sm top-1/2 -translate-y-1/2 h-4 w-4 text-muted-foreground`, pointer-events-none
- Input: `pl-[2.25rem]` (override left padding to clear icon), `pr-md` default or `pr-8` when clear button visible
- Clear button: `absolute right-sm`, appears only when input has value. Icon: `X` from lucide-react, `h-4 w-4`, `variant="ghost" size="icon" h-8 w-8`. `aria-label="Clear search"`. Clears input and removes `search` URL param.
- Debounce: 300ms from last keystroke before URL param updates (D-08)
- Controlled local state: `value` syncs to local state immediately for display; URL update is debounced
- Search query persists in URL — survives page navigation and refresh (D-08)
- `aria-label="Search plants"` on the `<Input>` element

**Layout position:**
```
[h1: "My Plants"]  [Add plant button]
[Search bar — full width]
[Room filter pills]
[Status filter pills]
[Sort dropdown — right-aligned]
[Plant grid]
```

### 2. Status Filter Pill Row (SRCH-02, D-10)

**Location:** Below room filter pills on `/plants` page.

**Pills (in order):** All | Overdue | Due today | Upcoming | Archived

**Spec:**
- Same `<Button size="sm">` pill pattern as existing `RoomFilter` component
- Row: `<div className="flex gap-sm overflow-x-auto pb-sm">` — matches room filter row exactly
- Active pill: colored per status palette above (see Color section)
- Inactive pill: `variant="outline"`
- Only one status filter active at a time (mutually exclusive). Clicking active pill deselects (returns to All).
- URL param: `status` — values: `overdue` | `due-today` | `upcoming` | `archived`
- Combinable with room filter (AND logic, D-11)
- When `status=archived`, the plant grid shows ONLY archived plants. The "Archived" pill is explicitly labeled — do not use color alone.

**Mobile:** Both filter rows are horizontally scrollable with `overflow-x-auto`. No wrapping.

### 3. Sort Dropdown (SRCH-03, D-12)

**Location:** Right-aligned, between the status filter row and the plant grid.

**Anatomy:**
```
[Status filter pills ←————————————] [Sort: Next watering ▾]
```

**Spec:**
- Trigger: `<Button variant="outline" size="sm">` with `ChevronsUpDown` or `ArrowUpDown` icon (`h-4 w-4`) on the right
- Button label: `"Sort: {active label}"` — e.g. "Sort: Next watering" (default), "Sort: Name (A-Z)", "Sort: Recently added"
- Menu: shadcn `<DropdownMenu>` with three items
- Active sort option shows a `Check` icon (`h-4 w-4`) on the left. Inactive options have no icon.
- URL param: `sort` — values: `next-watering` (default) | `name` | `recently-added`
- Default when no param: next watering date ascending

**DropdownMenu items:**
| Label | URL value | Icon when active |
|-------|-----------|-----------------|
| Next watering | `next-watering` | `Check h-4 w-4` |
| Name (A-Z) | `name` | `Check h-4 w-4` |
| Recently added | `recently-added` | `Check h-4 w-4` |

### 4. Unified Timeline Card (NOTE-02, D-01, D-03)

**Location:** Plant detail page — replaces the separate "Watering history" and "Notes" cards.

**Card anatomy:**
```
<Card>
  <CardHeader>
    <CardTitle>Timeline</CardTitle>
  </CardHeader>
  <CardContent>
    [Note add input row]         ← always visible at top
    [Separator]
    [Timeline entries — most recent first]
    [Load more button]           ← if totalCount > 20
  </CardContent>
</Card>
```

**Entry anatomy (note type):**
```
[Pencil icon in accent/10 circle]  [Note content (text-sm)]              [Kebab ⋮]
                                   [Timestamp — text-xs text-muted-foreground]
```

**Entry anatomy (watering type):**
```
[Droplets icon in blue/10 circle]  [Date "April 11, 2026" (text-sm font-medium)]  [Kebab ⋮]
                                   [Relative time "3 days ago" — text-xs muted]
                                   [optional note — text-sm muted]
```

**Type icon spec:**
- Note: `Pencil` from lucide-react, `h-4 w-4 text-accent`, container `h-8 w-8 rounded-full bg-accent/10 flex items-center justify-center shrink-0`
- Watering: `Droplets` from lucide-react, `h-4 w-4 text-blue-500`, container `h-8 w-8 rounded-full bg-blue-50 flex items-center justify-center shrink-0` (use `text-blue-500 bg-blue-50` — Tailwind v4 defaults; do not add a new CSS variable)
- Both containers use `shrink-0` to prevent compression in flex layouts

**Timestamp format:**
- Relative for recent (within 7 days): `"Today"`, `"Yesterday"`, `"3 days ago"` — date-fns `formatDistanceToNow` / `isToday` / `isYesterday`
- Absolute for older: `"April 4, 2026"` — `format(date, "MMMM d, yyyy")`
- Display: relative primary, absolute in tooltip (title attribute) — e.g. `<time dateTime={iso} title={absolute}>{relative}</time>`

**Load more:**
- Text button: `<Button variant="ghost" size="sm" className="w-full mt-sm">Load 20 more</Button>`
- Only renders when `totalCount > displayed entries`
- On click: calls `loadMoreTimeline` Server Action with current `skip` count

**Entry spacing:**
- Between entries: `<Separator />` between each entry (or `divide-y divide-border` on the list container)
- Entry internal padding: `py-sm` top and bottom

### 5. Inline Note Add (NOTE-01, D-04)

**Location:** Top of Timeline card content, above the separator and entry list.

**Anatomy:**
```
[Input: "Add a note..." placeholder]  [Add button]
```

**Spec:**
- `<Input placeholder="Add a note..." />` — full width, `text-sm`
- `<Button size="sm" variant="outline">Add</Button>` — right of input, in a `flex gap-sm` row
- Submit triggers on: Enter key (`onKeyDown` prevent default) OR clicking "Add" button
- Validation: non-empty after trim. If empty, do nothing (no error shown — silent guard).
- On submit: call `createNote` Server Action. Optimistic: append note entry to top of timeline immediately with pending state (grayed out, `opacity-60`). On success: entry becomes normal. On error: remove optimistic entry, show sonner toast.
- After success: clear input value
- `aria-label="Note text"` on the input

### 6. Inline Note Edit (NOTE-03, D-05)

**Trigger:** Kebab menu "Edit" on a note timeline entry.

**Interaction:**
1. Note entry switches from display mode to edit mode in place.
2. `<p className="text-sm">` becomes `<textarea className="text-sm w-full resize-none ...">` pre-filled with current content.
3. Textarea auto-sizes to content (use `rows={Math.max(2, lineCount)}`).
4. Below textarea: `<Button size="sm">Save</Button>` and `<Button size="sm" variant="ghost">Cancel</Button>` — in a `flex gap-sm` row.
5. Save on: "Save" button click OR `Ctrl+Enter` (keyboard shortcut).
6. Cancel on: "Cancel" button OR `Escape` key — restores display mode without saving.
7. On save: call `updateNote` Server Action. Optimistic: show updated content immediately. On error: revert to prior content and show toast.
8. `aria-label="Edit note"` on the textarea.

**Do not save on blur.** Blur without explicit Save/Cancel is treated as Cancel.

### 7. Delete Note (NOTE-03, D-05)

**Trigger:** Kebab menu "Delete" on a note timeline entry.

**Pattern:** shadcn `<AlertDialog>` — consistent with delete watering log from Phase 4.

**AlertDialog spec:**
- Title: "Delete note?"
- Body: "This note will be permanently removed from [Nickname]'s timeline."
- Destructive CTA: "Delete note" — `variant="destructive"`
- Dismiss: "Keep note" (ghost button)
- On confirm: call `deleteNote` Server Action. Optimistic: remove entry from timeline immediately. On error: restore entry and show toast.

### 8. Combined Empty State (Search / Filter Returns No Results)

**When:** Plant grid has 0 results after applying search query and/or filter.

**Anatomy:**
```
[Search icon in accent/10 circle — h-8 w-8]
[Heading: "No plants found"]
[Body: context-aware — see copy below]
[CTA — if applicable]
```

**Copy variants:**

| Condition | Heading | Body | CTA |
|-----------|---------|------|-----|
| Search query only | "No plants found" | "No plants match "{query}". Try a different name or spelling." | "Clear search" (clears search param) |
| Status filter only | "No plants found" | "No plants are {status label} right now." | "Show all plants" (clears status param) |
| Room + status | "No plants found" | "No {status label} plants in {room name}." | "Clear filters" (clears both params) |
| Archived filter, no archived plants | "No archived plants" | "Plants you archive will appear here." | None |

**Visual spec:**
- Icon container: `rounded-full bg-accent/10 p-lg mb-md` — matches existing empty state pattern from plants/page.tsx
- Icon: `Search` from lucide-react, `h-8 w-8 text-accent`
- Heading: `text-xl font-semibold`
- Body: `text-muted-foreground mt-sm`
- CTA: `<Button variant="outline" size="sm" className="mt-lg">` — clears relevant URL params on click

### 9. No Notes Yet (Empty Timeline — Notes Only)

When a plant has watering history but no notes, the note add input is shown at top and the timeline renders only watering entries. No dedicated "no notes" empty state is shown — the input area is the affordance.

When a plant has NO timeline entries at all (new plant, no watering logged, no notes):

**Anatomy:**
```
[Pencil icon in accent/10 circle]
[Body: "No history yet. Add a note or log a watering to get started."]
```

## Animation Contract

All animations use CSS transitions via Tailwind — no external animation library.

| Interaction | Animation | Duration | Notes |
|-------------|-----------|----------|-------|
| Note add (optimistic) | New entry appears at top of timeline with `opacity-60` | Instant | Fades to `opacity-100` on server confirm |
| Note delete (optimistic) | Entry fades out `opacity-0` before DOM removal | 200ms | Restore on error |
| Inline note edit transition | `<p>` → `<textarea>` — no animation, instant swap | — | Focus moves to textarea automatically |
| Save confirmation (edit) | Entry snaps back to display mode — no animation | — | — |
| Search debounce | Grid re-renders after 300ms — no explicit animation | — | React's natural re-render |
| Filter pill active state | No animation — instant state swap | — | Button variant change |
| Load more | New entries append below — no animation | — | — |
| Dialog open/close | shadcn default (Radix) | — | — |
| Toast | Sonner default | — | — |

**Reduce motion:** Optimistic opacity transitions must use `motion-safe:` prefix. `motion-safe:transition-opacity motion-safe:duration-200`.

## Responsive Layout Contract

| Viewport | Search + filter block | Plant grid | Timeline |
|----------|-----------------------|------------|----------|
| Mobile < 768px | Full width, stacked vertically. Filter rows scroll horizontally. Sort dropdown full width or right-aligned below. | 1-column | Single column, full width |
| Desktop ≥ 768px | Same layout — filters don't need to wrap at desktop widths. Sort dropdown right-aligned. | 2-column `grid-cols-2 gap-md` | Single column |

**Filter row mobile behavior:**
- Room pills: `overflow-x-auto pb-sm` — horizontal scroll, no wrapping
- Status pills: `overflow-x-auto pb-sm` — same horizontal scroll
- Both rows should NOT wrap on mobile. Horizontal scroll is the intentional behavior.

**Sort dropdown:**
- Positioned in a `flex justify-between items-center` row alongside a filter count/summary text if present, or right-aligned as `ml-auto` in the filter block.

**Search bar width:**
- Full width of the page content area (matches existing heading row layout)

Max content width: `max-w-5xl` (consistent with all other pages).
