## Color

All values from `src/app/globals.css` `@theme` block. Use CSS custom property names, not raw OKLCH values, in component code.

| Role | CSS Variable | OKLCH (light) | Usage |
|------|-------------|---------------|-------|
| Dominant (60%) | `--color-background` | oklch(97% 0.004 286) | Page background, auth layout background |
| Secondary (30%) | `--color-surface` | oklch(94% 0.006 286) | Auth form card background, onboarding banner card |
| Accent (10%) | `--color-accent` | oklch(62% 0.10 155) | Primary CTA button background, onboarding range button selected state, "Complete setup" return link, focus ring on inputs |
| Destructive | `--color-destructive` | oklch(55% 0.22 27) | Inline field validation errors, destructive action states |

Accent reserved for:
1. Primary CTA button fill ("Create account", "Sign in")
2. Onboarding range button — selected/active state border and text
3. "Complete setup" text link in nav/settings
4. Input and button focus ring (`--color-ring` maps to same green)
5. Onboarding banner soft green gradient background (use `--color-accent` at 15% opacity)

The accent color is the plant-inspired green (oklch 62% chroma 0.10 hue 155). It must not be applied to informational text, decorative borders, or secondary actions.

## Screen Inventory

### Screen 1: Login Page (`/login`)

**Layout:** Centered single-column card. Auth route group layout (`src/app/(auth)/layout.tsx`) provides the centering shell.

**Card dimensions:** max-width 400px, padding lg (24px).

**Elements (top to bottom):**
1. App wordmark / logo — "Plant Minder" in heading weight, accent color leaf icon (Lucide `Leaf`)
2. Page heading — "Sign in to your account" (heading, 24px/600)
3. Email field — Label + Input + inline error slot
4. Password field — Label + Input + password visibility toggle (Lucide `Eye`/`EyeOff`, 44px hit target) + inline error slot
5. Primary CTA button — "Sign in" (full width, accent fill)
6. Server error toast — rendered by Sonner on auth failure (see Copywriting)
7. Cross-link — "Don't have an account? Sign up" (body 16px, link text in accent color)

**Loading state:** Primary button shows Lucide `Loader2` spinner (animate-spin) and is disabled while the signIn call is in-flight. No skeleton — the card itself is the shell.

### Screen 2: Register Page (`/register`)

**Layout:** Identical centering shell to login.

**Card dimensions:** max-width 400px, padding lg (24px).

**Elements (top to bottom):**
1. App wordmark / logo — same as login
2. Page heading — "Create your account" (heading, 24px/600)
3. Email field — Label + Input + inline error slot
4. Password field — Label + Input + visibility toggle + inline error slot
5. Confirm password field — Label + Input + visibility toggle + inline error slot
6. Primary CTA button — "Create account" (full width, accent fill)
7. Server error toast — rendered by Sonner on duplicate email or network failure
8. Cross-link — "Already have an account? Sign in" (body 16px, link in accent)

**Loading state:** Same spinner pattern as login. All three fields disabled while action is in-flight.

### Screen 3: Dashboard with Onboarding Banner (`/dashboard` — first visit only)

**Layout:** Full-width dashboard shell. Onboarding banner spans the full content width at the top, above the main dashboard content area. Dashboard content (empty state) is visible below the banner — not blocked.

**Banner dimensions:** full content width, padding md (16px) horizontal, lg (24px) vertical. Background: `--color-accent` at 15% opacity (soft green wash). Border: 1px solid `--color-accent` at 30% opacity. Border-radius: uses `--radius-lg`.

**Banner elements (left to right, single row on desktop / stacked on mobile):**
1. Plant icon — Lucide `Leaf` in accent color, 24px
2. Banner heading — "Welcome to Plant Minder" (label weight 500, 16px)
3. Banner body — "How many plants are you tracking?" (body 16px/400)
4. Range buttons row — 4 buttons: "1–5", "6–15", "16–30", "30+" (Button variant `outline`, 44px height; selected state: accent border + accent text)
5. Dismiss button — "×" icon button (Lucide `X`, 44px hit target, `--color-muted` color), positioned top-right of the card

**Post-selection state:** After user taps a range button, the button row is replaced by a brief confirmation line "Got it — your tips are personalized." (muted text, 14px). Banner then collapses via CSS max-height transition (300ms ease-out) and is removed from the DOM.

**Dismiss (without completing):** Dismiss button unmounts the banner via React state — no server call, no DB write. Banner slides up via same collapse animation.

**Return path:** A "Complete setup" text link is placed in the authenticated nav (top-right user menu or settings page) — accent color, 14px label weight. It remounts the banner by setting a local UI state flag (does not re-query DB on each visit after completion).

Note: Login errors use a single generic message for wrong credentials — do not distinguish "email not found" from "wrong password" (security best practice). Source: Claude's Discretion, security default.

### Auth Error Handling Policy (Claude's Discretion — applied)

- **Failed login:** Generic error only — "Incorrect email or password." Never reveal whether the email exists.
- **Rate limiting:** Not implemented in v1. Out of scope.
- **Session expiry:** If `auth()` returns null on a protected route, `proxy.ts` redirects to `/login` transparently. No in-app re-auth dialog.
- **Password requirements:** Minimum 6 characters (matches existing Zod schema in `auth.ts`). No complexity rules for v1.

## Accessibility Contract

Requirements UIAX-02 and UIAX-03 are Phase 7 targets but must not be violated in Phase 2 implementation:

- All form inputs must have an associated `<label>` with matching `htmlFor`/`id`
- Inline error messages must use `aria-describedby` linking input to error element
- Error state must not rely on color alone — error text must be present alongside red border
- Password visibility toggle must have `aria-label="Show password"` / `aria-label="Hide password"`
- Onboarding dismiss button must have `aria-label="Dismiss setup banner"`
- Range buttons must be keyboard-focusable and activatable via Enter/Space
- Toast notifications (Sonner) must have `role="status"` or `role="alert"` — Sonner handles this by default
