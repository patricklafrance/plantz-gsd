**4. Push schema and backfill Reminder records:**

Run: `npx prisma db push`

This applies the `remindersEnabled` field on User and `@@unique([plantId, userId])` on Reminder.

After the push succeeds, run a one-time backfill to create Reminder records for existing plants that do not have one. Execute via `npx prisma db execute`:

```sql
INSERT INTO "Reminder" (id, "plantId", "userId", enabled, "snoozedUntil", "createdAt", "updatedAt")
SELECT gen_random_uuid(), p.id, p."userId", true, NULL, NOW(), NOW()
FROM "Plant" p
WHERE NOT EXISTS (
  SELECT 1 FROM "Reminder" r WHERE r."plantId" = p.id AND r."userId" = p."userId"
);
```

If `npx prisma db execute` is not available or fails, create a temporary `prisma/backfill-reminders.ts` script that runs the raw SQL via `db.$executeRawUnsafe(...)` and execute it with `npx tsx prisma/backfill-reminders.ts`. Delete the script after execution.
