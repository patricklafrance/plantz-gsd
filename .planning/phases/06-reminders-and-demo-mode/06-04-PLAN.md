Modify the `handleRangeSelect` function to also call `seedStarterPlants` when `seedStarters` is true:

```typescript
async function handleRangeSelect(range: string) {
  setSelectedRange(range);
  setIsCompleting(true);

  // Fire both actions concurrently
  const [onboardingResult] = await Promise.all([
    completeOnboarding({ plantCountRange: range }),
    seedStarters ? seedStarterPlants() : Promise.resolve(null),
  ]);

  setIsCompleting(false);

  if (onboardingResult && "error" in onboardingResult) {
    toast.error("Something went wrong. Please try again.");
    setSelectedRange(null);
    return;
  }

  setIsCompleted(true);
  setTimeout(() => setDismissed(true), 1500);
}
```

| Boundary | Description |
|----------|-------------|
| /demo route -> signIn | Unauthenticated user triggers real NextAuth sign-in as demo user |
| Onboarding -> seedStarterPlants | Authenticated user seeds plants; demo guard prevents demo user from seeding |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-15 | Spoofing | /demo route | accept | Demo sign-in uses real credentials and real NextAuth flow; session is legitimate but flagged isDemo |
| T-06-16 | Tampering | seedStarterPlants | mitigate | Server Action checks `session.user.isDemo` and rejects; only real users can seed |
| T-06-17 | Elevation of Privilege | Demo user accessing /preferences | mitigate | All preference toggle actions check isDemo and return error; forms are disabled in UI |
| T-06-18 | Denial of Service | /demo hit repeatedly | accept | Each /demo hit signs in as the same demo user; no new DB rows created; minimal impact |
</threat_model>
