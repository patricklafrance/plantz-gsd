## Spacing Scale

All spacing tokens already declared in `src/app/globals.css` under `@theme`. Use these CSS custom properties — do not introduce new spacing values.

| Token | Value | Usage |
|-------|-------|-------|
| `--spacing-xs` | 4px | Icon gaps, badge padding, tight inline spacing |
| `--spacing-sm` | 8px | Plant card internal gaps, form field label-to-input gap |
| `--spacing-md` | 16px | Default element spacing, nav padding, modal content padding |
| `--spacing-lg` | 24px | Section padding, card internal padding, page vertical rhythm |
| `--spacing-xl` | 32px | Layout column gaps, between major sections |
| `--spacing-2xl` | 48px | Empty state vertical padding |
| `--spacing-3xl` | 64px | Full-page empty state centering (matches dashboard pattern) |

**Tailwind usage:** `px-md`, `py-lg`, `gap-sm`, `space-y-lg` — these map to the `--spacing-*` tokens via shadcn's Tailwind v4 CSS-first config.

**Exceptions:**

- Touch targets for icon-only buttons (archive/delete on mobile): minimum 44px hit area via `min-h-11 min-w-11`. Visual size stays at 20px icon; padding fills the gap.
- Catalog grid cards: `p-sm` (8px) internal padding to fit more cards in the modal viewport.

## Color

All color tokens defined in `src/app/globals.css`. No new tokens introduced this phase.

| Role | Token | Light Value | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `--color-background` | oklch(97% 0.004 286) — near-white warm gray | Page background, modal overlay fill |
| Secondary (30%) | `--color-surface` / `--card` | oklch(94% 0.006 286) — slightly darker warm gray | Plant cards, room cards, detail card sections, sidebar nav |
| Accent (10%) | `--color-accent` | oklch(62% 0.10 155) — muted green | See reserved-for list below |
| Destructive | `--color-destructive` | oklch(55% 0.22 27) — red-orange | Delete plant button, AlertDialog destructive trigger only |

**Accent reserved for:**
1. "Add plant" primary CTA button
2. Active room filter pill/tab indicator
3. Plant card icon container background (`bg-accent/10` with accent-colored leaf icon) — matches dashboard empty state pattern
4. Catalog selection highlight (selected plant card border or background tint)
5. Link hover states in nav (matches existing `text-accent hover:underline` pattern)

**Accent is NOT used for:** Edit button, archive button, room preset chips, modal close button, form submit inside edit modal (use `--primary` for those).

**Watering status colors (semantic, this phase scoped to plant detail Status card only):**

| Status | Color approach | Rationale |
|--------|---------------|-----------|
| Overdue | `text-destructive` | Danger signal — plant needs immediate attention |
| Due today | `text-accent` (amber/orange not available; reuse green accent with copy support) | Phase 4 will refine; for now use accent color + "Due today" label |
| Upcoming | `text-muted-foreground` | No urgency |
| Recently watered | `text-muted-foreground` | No urgency |

Note: Full urgency color system is a Phase 4 concern. In Phase 3, the Status card only needs to display the calculated next watering date without urgency-tier color coding beyond overdue vs not-overdue.

## Component Inventory

### Plant Card (`plant-card.tsx`)

- Built on shadcn `Card` (uses `--card` surface color)
- Layout: horizontal on desktop (`flex items-center gap-md`), full-width on mobile
- Left: 40px square icon container, `rounded-md bg-accent/10`, Lucide `Leaf` icon at 20px in `text-accent`
- Body: plant nickname (`text-base font-semibold`), species name (`text-sm text-muted-foreground`), room label (`text-xs text-muted-foreground`)
- Right: watering status chip (Badge component) — text only for Phase 3; urgency colors in Phase 4
- Hover state: `hover:shadow-sm hover:border-accent/40` on the Card
- Click target: entire card navigates to `/plants/[id]`

### Catalog Grid Card (inside add-plant modal, step 1)

- Smaller than plant card: fixed `w-full` in a 2-column or 3-column grid
- Layout: icon centered top, common name below (`text-sm font-medium`), species name (`text-xs text-muted-foreground`)
- Selected state: `ring-2 ring-accent bg-accent/5`
- "Custom plant" option: rendered as a card at the bottom of the grid with a `Plus` icon and "Custom plant" label

### Room Filter Pill Bar

- Horizontal scrollable `flex gap-sm` above plant grid
- Each pill: `Button` with `variant="outline"` at default size; active pill uses `variant="default"` (primary fill)
- Pills: "All" first, then room names in creation order
- Mobile: `overflow-x-auto` with `pb-sm` to allow horizontal scroll without layout break
- Do NOT use shadcn Tabs here — plain Button pills give simpler active state control

### Add / Edit Plant Modal

- shadcn `Dialog` with max-width `max-w-lg` on desktop, full-screen (`w-full h-full rounded-none`) on mobile at breakpoint `sm:`
- Step 1 (Catalog): `DialogTitle` = "Choose a plant", search `Input` at top, category label (`text-sm font-semibold`), 2-column catalog card grid below, "Custom plant" card at bottom
- Step 2 (Form): `DialogTitle` = "Add plant details" (add) or "Edit plant" (edit), `Form` with fields: Nickname (Input, required), Species (Input, auto-filled, editable), Room (Select), Watering interval (Input type="number", suffix "days")
- Step navigation: back arrow (`ChevronLeft` + "Back to catalog") at top left of step 2; not shown in edit modal (no catalog step)
- Submit CTA: "Add plant" (add flow) or "Save changes" (edit flow)
- Cancel: `DialogClose` text link "Cancel" left of CTA, not a destructive button

### Plant Detail Page (`/plants/[id]`)

- Four card sections stacked vertically with `space-y-lg`:
  1. **Status card** — "Next watering" date + "X days" countdown or "Due today" / "Overdue" badge
  2. **Care info card** — Species, watering interval, light requirement (icon + label)
  3. **History card** — Heading "Watering history", empty state message for Phase 3 ("No waterings logged yet. History will appear here after Phase 4.")
  4. **Notes section** — Heading "Notes", empty state for Phase 3 ("Notes coming in Phase 5.")
- Action row below page title: "Edit" (`Button variant="outline"` with `Pencil` icon), "Archive" (`Button variant="outline"` with `Archive` icon), "Delete" (`Button variant="outline" className="text-destructive hover:bg-destructive/10"` with `Trash2` icon)

### Room Management Page (`/rooms`)

- Page heading "Rooms" + "New room" button (accent primary)
- Quick-create preset chips: 6 preset names rendered as `Badge` or small outline `Button`, clicking one creates the room inline (no modal needed for preset creation)
- Custom room: "New room" button opens a small `Dialog` (single Input for name + "Create room" submit)
- Room cards: `Card` with room name (`text-base font-semibold`), plant count (`text-sm text-muted-foreground`), edit icon (`Pencil`), delete icon (`Trash2`)
- Overflow menu pattern: on mobile, the edit/delete icons collapse into a `...` `DropdownMenu` (shadcn DropdownMenu not yet installed — add it if overflow needed, or keep icons visible at all breakpoints with `min-w` protection)

### Room Detail Page (`/rooms/[id]`)

- Header: room name (Heading size), plant count (`text-muted-foreground`), "Edit room" link
- Grid: responsive plant card grid — 2 columns on mobile (`grid-cols-2`), 3 columns on desktop (`grid-cols-3`), `gap-md`
- Empty state: see Copywriting Contract below

## Interaction Contracts

### Two-Step Add Plant Modal

1. Trigger: "Add plant" button (nav or collection page). Opens Dialog with step = "catalog".
2. Catalog step: Search bar filters catalog cards in real time (client-side). Category label groups visible. User clicks a catalog card → `selectedProfile` set, step transitions to "form". User clicks "Custom plant" → `selectedProfile = null`, step transitions to "form".
3. Form step: Fields pre-filled from `selectedProfile` if present. All fields editable. "Back to catalog" chevron resets to step = "catalog" (does not clear nickname if user typed it).
4. Submit: calls `createPlant` Server Action. On success: closes dialog, shows "Plant added." toast, revalidates `/plants` and `/dashboard`. On error: shows inline form error.
5. Close (X or Escape): resets both step and `selectedProfile` to initial state.

### Archive Plant

1. Trigger: "Archive" button on plant detail page.
2. No confirmation dialog — instant action.
3. Calls `archivePlant` Server Action. Sets `archivedAt` timestamp.
4. On success: navigate to `/plants` (or dashboard), show Sonner toast "Plant archived." with "Undo" action.
5. Undo: calls `unarchivePlant` Server Action (clears `archivedAt`), dismisses toast, shows "Archive undone." toast.
6. Toast timeout: 5 seconds before undo option disappears.

### Delete Plant

1. Trigger: "Delete plant" button on plant detail page.
2. Opens shadcn `AlertDialog` with title, body (with plant nickname interpolated), destructive CTA, and Cancel.
3. Cancel: closes dialog, no action taken.
4. Confirm: calls `deletePlant` Server Action (hard delete via Prisma cascade). On success: navigate to `/plants`, show "Plant deleted." toast (no undo).

### Room Filter

1. Pill bar above plant grid on `/plants` page and on collection within dashboard (Phase 4).
2. Active pill state managed via URL search param `?room={roomId}` — allows browser back/forward and shareable URLs.
3. "All" pill clears the `room` param.
4. Server Component reads `searchParams.room`, filters plant query by `roomId`.

### Room Deletion with Plants

1. Trigger: Delete icon on room card.
2. Opens AlertDialog. Body checks if room has plants. If plants exist: show warning "X plants will become unassigned. They will still appear in your collection." If no plants: standard "Delete this room?" copy.
3. Confirm: calls `deleteRoom` Server Action. Nullifies `roomId` on all plants in room (set `roomId = null`). Revalidates `/rooms` and `/plants`.
